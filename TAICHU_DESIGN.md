# 太初前端设计说明

> 更新日期：2026-06-30
> 强制适用范围：太初前端视觉系统、页面布局、组件设计、交互动效、Tailwind token、shadcn/ui 二次封装和所有用户可见前端文案。

本文档是太初前端设计的强制规则源。涉及 `web/` 下页面、组件、样式、交互、动效、前端文案或视觉方案的创建、修改、评审和讨论时，必须先读取并遵循本文档。该要求同样适用于任务包执行、自动前端开发、自动修复、自动重构和前端 review，不以用户是否显式提出“使用设计规则”或“使用组件库”为前提。不得使用其他视觉规则覆盖本文档，也不得将本文档仅作为可选参考。

本文档只定义前端视觉与交互规则，不改变太初的产品边界：太初仍然是面向单本玄幻小说的个人 AI 写作助手，不支持多小说管理、多租户后台或通用内容平台形态。

## 1. 设计方向

太初前端统一采用“奶油纸张画布 + 深墨绿锚点 + 薰衣草高亮 + 编辑出版感字体”的编辑科技风格。界面应像一份为作者工作的精致写作出版工具：温暖、清晰、有纸张质感，但仍保留现代产品的效率、结构和可操作性。

核心判断：

- 主画布以奶油纸色为基础，避免冷白、纯黑或高饱和底色成为默认页面背景。
- 深墨绿用于品牌锚点、重要区段、徽标和强调带，承担视觉重量。
- 薰衣草色只用于主操作、高亮线、标题重点和少量提示，不铺成大面积背景。
- 深墨色用于正文、边框、深色展示区和结构分隔，替代纯黑作为主要暗色。
- 大字号衬线标题建立“编辑出版感”，功能文字使用清晰无衬线字体。
- 页面视觉以排版、边框、留白、圆角和少量插画/产品预览承载，不靠霓虹、炫光或游戏化装饰。

设计目标：

- 写作场景要安静，支持长时间阅读、编辑、检索和 AI 协作。
- 资料、知识、章节、灵感和审查结果要像可归档的编辑材料，而不是普通后台数据表。
- 入口、功能页、编辑器和资料页使用同一套色彩与字体，不再维护互相割裂的视觉系统。
- 所有用户可见文案必须是中文。内部组件名、token 名、接口名和文件路径可以使用英文，但不能直接暴露为界面文字。

## 2. 颜色 Token

所有颜色必须通过 CSS variables 或 Tailwind token 使用。允许保留旧 token 作为迁移期兼容，但新开发、新重构和新组件必须优先使用本节 token。

```css
:root {
  --tc-cream-paper: #ffffeb;
  --tc-midnight-ink: #1a1a1a;
  --tc-pure-black: #000000;
  --tc-white: #ffffff;
  --tc-stone-mist: #e4e4d0;
  --tc-graphite-veil: #8a8a80;
  --tc-smoke: #5f5f59;
  --tc-charcoal: #222222;
  --tc-charcoal-mist: #333333;
  --tc-deep-forest-teal: #034f46;
  --tc-lavender-whisper: #f0d7ff;
  --tc-amber-pulse: #ffa946;
}
```

使用规则：

- `--tc-cream-paper` 是默认页面画布、浅色内容区和普通卡片底色。
- `--tc-midnight-ink` 是主要文字、深色区块、强边框和暗色控件底色。
- `--tc-pure-black` 只用于装饰图形、局部阴影或必须贴合素材的 SVG 填充，不作为常规文字或页面背景。
- `--tc-white` 用于深色区块文字、浮动导航表面和反色按钮文字。
- `--tc-stone-mist` 用于奶油纸画布上的弱边框、弱分隔和轻量二级表面。
- `--tc-graphite-veil` 用于次级标题、说明文字、控件描边和双色标题的弱势部分。
- `--tc-smoke` 用于低强调说明、时间、来源、状态补充等三级信息。
- `--tc-charcoal` 和 `--tc-charcoal-mist` 用于深色控件层级、导航边框和暗色区块内部分隔，不升级为主按钮色。
- `--tc-deep-forest-teal` 用于品牌条、重要状态带、资料归档区、徽标和深色强调区。
- `--tc-lavender-whisper` 用于每屏唯一主操作、标题下划高亮、选区提示和轻量标记。
- `--tc-amber-pulse` 只用于少量温暖插画、警示点、徽标细节或局部标记。

禁止：

- 禁止新增蓝、红、霓虹绿等额外主色作为按钮、导航或状态体系。
- 禁止让薰衣草色成为大面积页面背景。
- 禁止在薰衣草色上使用白色或浅色文字，必须使用深墨色文字。
- 禁止用纯黑作为正文或主背景默认色。

## 3. 字体与排版

### 3.1 展示标题

展示标题使用 EB Garamond 与中文衬线 fallback。它承担品牌气质、章节感和编辑出版感。

```css
--tc-font-display: "EB Garamond", "Noto Serif SC", "Source Han Serif SC", "Songti SC", "SimSun", serif;
```

适用范围：

- 首页和功能入口的主标题。
- 页面级标题。
- 章节、知识、导出预览的展示标题。
- 重要空状态标题和内容集合标题。

规则：

- 展示标题建议使用 48px、64px、80px、96px 或 120px，移动端可降到 36px 或 40px。
- 字重以 400 为主，不用粗黑标题压迫页面。
- `letter-spacing` 必须为 `0`，不得使用负字距。
- 通过字号、行高、留白、双色标题建立层级，不靠压缩字距制造视觉冲击。
- 中文长标题必须优先保证可读性，不能为了造型造成断词、遮挡或溢出。

### 3.2 功能文字

UI、正文、导航、按钮、标签、表单和说明文字使用 Figtree 与中文无衬线 fallback。

```css
--tc-font-ui: "Figtree", "Inter", "PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif;
```

适用范围：

- 导航。
- 按钮。
- 表单。
- 卡片正文。
- 资料列表。
- 标签与徽标。
- AI 状态文案。
- 页面说明和帮助文字。

规则：

- 常规正文使用 16px，行高 1.3 到 1.6；长篇写作正文可放宽到 1.7 到 1.9。
- 导航、标签和按钮以 14px 到 16px 为主。
- 字重常用 400、500、600，只有局部强调允许 700。
- 用户可见的命令、按钮、报错、状态、提示和表单标签必须使用中文。

### 3.3 等宽读数

技术读数可以使用等宽字体，但必须配中文解释。

```css
--tc-font-mono: "Geist Mono", "JetBrains Mono", "SFMono-Regular", Consolas, monospace;
```

适用范围：

- 来源编号。
- 时间戳。
- 字数读数。
- 模型状态。
- 快捷键。
- 文件或接口示例。

规则：

- 等宽字体不能用于长段中文正文。
- 不直接显示内部字段名，例如 `source_ref_count` 应显示为“来源 3 条”。

## 4. 空间、圆角与边框

基础单位为 8px。布局应舒展但不松散，信息密度服务写作效率。

```css
:root {
  --tc-space-8: 8px;
  --tc-space-16: 16px;
  --tc-space-24: 24px;
  --tc-space-32: 32px;
  --tc-space-40: 40px;
  --tc-space-48: 48px;
  --tc-space-64: 64px;
  --tc-space-80: 80px;
  --tc-space-96: 96px;
  --tc-page-max-width: 1200px;
  --tc-radius-control: 14px;
  --tc-radius-card: 32px;
  --tc-radius-badge: 1000px;
  --tc-radius-image: 40px;
  --tc-radius-small: 8px;
}
```

规则：

- 页面主体内容最大宽度默认 1200px，特殊编辑视图可按功能扩展。
- 大区块垂直间距使用 64px 到 80px。
- 普通卡片内边距默认 24px 到 32px。
- 按钮、输入框、导航容器使用 14px 圆角。
- 功能卡、资料卡、展示卡使用 32px 圆角。
- 小标签使用 8px 或全圆角。
- 边框优先使用 1px 细线，浅底上用 `--tc-stone-mist` 或 `--tc-midnight-ink`，深底上用 `--tc-charcoal-mist` 或 `--tc-white`。
- 阴影必须克制，不能形成浮夸卡片堆叠。

## 5. 页面模式

### 5.1 奶油纸主页面

适用于 `/`、`/home`、知识列表、灵感列表、设置和普通功能入口。

规则：

- 页面背景使用 `--tc-cream-paper`。
- 标题优先使用展示字体，正文和说明使用 UI 字体。
- 主要内容放在 1200px 容器内，避免全屏铺满信息。
- 可使用白色或奶油纸卡片承载功能入口，边框用 `--tc-stone-mist`。
- 每屏只保留一个薰衣草主按钮，其余操作使用描边按钮或文字按钮。
- 可使用深墨绿小条、徽标或标题强调建立品牌锚点。

### 5.2 深墨色展示区

适用于重点功能演示、AI 能力展示、导出预览外壳和需要反差的说明区。

规则：

- 区块背景使用 `--tc-midnight-ink`。
- 文字使用 `--tc-white`，次级文字使用 `--tc-graphite-veil` 或低透明白。
- 区块必须与奶油纸区交替出现，不能连续堆叠多个深色区。
- 深色区可嵌入产品预览、编辑器片段、资料卡缩略图或简洁插画。
- 按钮使用白色描边或薰衣草主按钮，仍遵守每屏唯一主操作。

### 5.3 深墨绿强调带

适用于状态公告、重要提示、资料归档完成、章节里程碑、知识库摘要和可信来源展示。

规则：

- 背景使用 `--tc-deep-forest-teal`。
- 文字使用 `--tc-white`。
- 内容应简短，避免变成复杂工具区。
- 可承载一行说明、少量指标、来源数量或完成状态。

### 5.4 编辑与写作页面

写作页面应保持奶油纸与编辑出版感，但必须优先保障输入效率。

规则：

- 编辑器正文区域使用浅色纸张表面，正文使用中文可读字体和足够行高。
- 章节树、工具栏、AI 辅助区使用同一套奶油纸/白色/深墨边框语言，不另起深灰控制台体系。
- 当前章节、保存状态、AI 状态和来源数量以小标签、徽标或细线提示呈现。
- AI 结果可以作为圆角卡片出现在右侧或下方，不能遮挡正在输入的正文。
- 长文本编辑区域不要使用大面积薰衣草或墨绿底色。

### 5.5 资料与知识页面

资料页要像编辑档案库，而不是后台表格。

规则：

- 卡片标题可使用展示字体或较克制的 UI 字体，正文使用 UI 字体。
- 来源、状态、时间和分类使用小徽标呈现。
- 已确认资料可使用深墨绿徽标；待处理资料可使用薰衣草或琥珀小标记。
- 详情页可以使用 32px 圆角卡片与细边框，正文排版应适合阅读。

## 6. 组件规则

### 6.1 按钮

主按钮：

- 背景 `--tc-lavender-whisper`。
- 文字 `--tc-midnight-ink`。
- 1px `--tc-midnight-ink` 边框。
- 圆角 14px。
- 内边距建议 10px 18px。
- 每个视图只允许一个主要薰衣草按钮。

次级按钮：

- 透明或白色底。
- 1px 深墨色或石雾色边框。
- 文字使用深墨色。
- 深色区中使用白色描边和白色文字。

图标按钮：

- 优先使用 `lucide-react` 图标。
- 必须有中文可访问名称和 tooltip。
- 不能只依赖颜色表达含义。

禁止：

- 禁止多种高饱和按钮并列。
- 禁止把薰衣草按钮作为普通列表操作反复出现。
- 禁止使用英文示例文案作为按钮文字。

### 6.2 导航

导航应像轻量出版工具的悬浮栏。

规则：

- 背景优先使用 `--tc-white`。
- 边框使用 1px `--tc-stone-mist`。
- 圆角 14px。
- 品牌、主入口、当前模块和必要操作清晰可见。
- 导航文字使用 UI 字体 14px 到 16px。
- 移动端导航必须可折叠，不能挤压正文或编辑器。

### 6.3 卡片

功能卡、资料卡、AI 结果卡和入口卡统一使用编辑卡片语言。

规则：

- 背景使用 `--tc-cream-paper` 或 `--tc-white`。
- 圆角 32px，紧凑小卡可降到 14px。
- 边框使用 1px `--tc-stone-mist`。
- 内边距默认 24px 到 32px。
- 标题使用 UI 字体 20px 到 24px；展示型卡片可使用展示字体。
- 正文使用 UI 字体 16px，行高 1.3 到 1.6。
- 可嵌入截图、稿件片段、插画或小型图表，但必须服务内容。

### 6.4 标签与徽标

标签用于类型、状态、来源、章节和阶段。

规则：

- 默认全圆角或 8px 小圆角。
- 字号 12px 到 14px。
- 深墨绿填充用于已确认、已归档、来源可信等状态。
- 薰衣草填充用于可操作、高亮、选区相关状态。
- 琥珀色只用于少量提醒或待处理状态。
- 标签必须有中文文字，不允许只用色块。

### 6.5 输入框与表单

规则：

- 背景使用白色或奶油纸色。
- 边框使用 `--tc-stone-mist`，聚焦时使用深墨色或薰衣草描边。
- 圆角 14px。
- 标签、占位和错误提示使用中文。
- 错误状态以文字、图标和边框说明，不使用大面积红色铺底。

### 6.6 AI 状态与结果

AI 应表现为写作工具的辅助能力，而不是视觉主角。

规则：

- 生成中、检索中、整理来源等状态使用小徽标、细进度条或短文本。
- AI 结果卡使用普通卡片规则，必要时用薰衣草细线标记来源于 AI。
- 引用来源必须显示为“来源”“章节”“选区”“手动确认”等中文。
- 操作项使用中文动词，例如“插入正文”“保存为灵感”“转为设定”“丢弃”。
- 不使用全屏 loading，不使用大面积发光效果。

## 7. 动效规则

动效只用于说明层级、状态和反馈，不作为装饰主角。

规则：

- 常规 hover、focus、popover 出现使用 120ms 到 200ms。
- 卡片进入可使用 180ms 到 260ms 的淡入和轻微位移。
- 主要页面切换避免强动效，优先保持写作连续性。
- 薰衣草高亮可做轻微淡入，不做持续闪烁。
- 尊重 `prefers-reduced-motion`，关闭非必要位移和循环动画。

禁止：

- 禁止持续旋转装饰。
- 禁止大面积渐变流动背景。
- 禁止强 glow、粒子爆炸或赛博效果。
- 禁止动效干扰输入、选中文本和阅读。

## 8. 图像与插画

太初可以使用视觉资产，但应服务写作与资料组织。

规则：

- 优先使用产品预览、稿件片段、资料卡缩略图、章节结构示意和扁平插画。
- 插画色彩以奶油纸、深墨绿、薰衣草、琥珀为主。
- 不使用生活方式摄影作为主要视觉语言。
- 不使用与写作无关的三方 logo 墙或下载平台展示。
- 首页和功能入口应能第一屏识别“太初”与“单本玄幻小说写作助手”的关系。

## 9. Tailwind 与 shadcn/ui 映射

建议在 `web/src/app/globals.css` 或 `web/src/styles/tokens.css` 中声明新 token，并逐步把 shadcn/ui 主题变量映射到当前风格。

```css
:root {
  --background: var(--tc-cream-paper);
  --foreground: var(--tc-midnight-ink);
  --card: var(--tc-white);
  --card-foreground: var(--tc-midnight-ink);
  --popover: var(--tc-white);
  --popover-foreground: var(--tc-midnight-ink);
  --primary: var(--tc-lavender-whisper);
  --primary-foreground: var(--tc-midnight-ink);
  --secondary: var(--tc-stone-mist);
  --secondary-foreground: var(--tc-midnight-ink);
  --muted: var(--tc-stone-mist);
  --muted-foreground: var(--tc-smoke);
  --accent: var(--tc-deep-forest-teal);
  --accent-foreground: var(--tc-white);
  --border: var(--tc-stone-mist);
  --input: var(--tc-stone-mist);
  --ring: var(--tc-lavender-whisper);
}
```

Tailwind v4 映射建议：

```css
@theme {
  --color-tc-cream-paper: #ffffeb;
  --color-tc-midnight-ink: #1a1a1a;
  --color-tc-stone-mist: #e4e4d0;
  --color-tc-graphite-veil: #8a8a80;
  --color-tc-smoke: #5f5f59;
  --color-tc-deep-forest-teal: #034f46;
  --color-tc-lavender-whisper: #f0d7ff;
  --color-tc-amber-pulse: #ffa946;
  --font-tc-display: "EB Garamond", "Noto Serif SC", serif;
  --font-tc-ui: "Figtree", "Inter", "PingFang SC", sans-serif;
}
```

迁移规则：

- 新组件不得继续扩展旧视觉 token。
- 旧 token 可以作为兼容层保留，但新增样式必须优先使用当前 token。
- 外部 registry 组件落地后必须改为太初 token、中文文案和本项目目录结构。
- 不允许保留外部组件原始品牌色、英文示例文案或默认营销布局。

## 10. 可访问性与中文文案

规则：

- 所有按钮、图标按钮、菜单项和表单控件必须有中文可访问名称。
- 文本和背景对比度必须满足长时间阅读。
- 标签不能只靠颜色表达状态，必须配文字。
- 键盘用户必须能操作导航、命令面板、AI 操作、表单和弹层。
- 用户可见文案必须面向作者和写作任务，不直接暴露内部字段名、接口名或枚举值。

文案示例：

- 使用“保存为灵感”，不要显示 `save_to_inbox`。
- 使用“来源 3 条”，不要显示 `source_ref_count: 3`。
- 使用“正在整理章节摘要”，不要显示 `generating_summary`。
- 使用“转为设定”，不要显示 `convert_to_pending_fact`。

## 11. 明确禁止项

太初前端不追求以下方向：

- 游戏 UI。
- 赛博霓虹。
- 随机星空、粒子入口或观测站隐喻。
- 普通后台管理系统。
- 多色高饱和按钮体系。
- 大面积薰衣草背景。
- 大面积纯黑背景。
- 强光效、玻璃拟态、炫光卡片。
- 只展示营销口号而没有实际写作工作入口。
- 把英文内部标识直接作为用户可见文案。

最终判断标准：

```text
打开太初时，用户看到的是温暖、精致、可信的中文写作出版工具。
开始写作时，界面优先服务章节编辑、资料组织和 AI 协作。
沉淀内容时，资料像可归档的编辑稿件，而不是散乱的数据卡片。
```
