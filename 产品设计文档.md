# AI 写作助手产品设计文档（v1.0-MVP 草案）

> 更新日期：2026-06-24

## 一、产品定位

个人 AI 写作助手，专注于**玄幻小说写作**场景。以知识库为核心，提供沉浸式写作体验。

## 二、用户交互模型

### 核心体验：统一聊天窗口 + 上下文操作按钮

```
  ┌──────────────────────────────────────────────────────┐
  │  小说创作助手                                        │
  │                                                      │
  │  💬 你：我想写一个主角被退婚的开头                  │
  │  🤖 AI：好创意！可以从「订婚宴现场收到退婚书」开始  │
  │  💬 你：帮我写一段这个场景(如果没说字应该弹窗给一个多少字)      │
  │  🤖 AI：（生成了一段XX字的开篇）                    │
  │                                                      │
  │  [选中文字/其他操作] ──┐                                     │
  │              │  [📝续写] [🎨润色] [⚡相关灵感生成]        │
  │              │  [📏扩写] [📏缩写] [🔍查错]         │
  │              │  [🔎搜设定] [📊检查设定]             │
  │              └─────────────────────────────────────  │
  └──────────────────────────────────────────────────────┘
```

### 辅助页面：独立管理视图

| 页面 | 用途 | 交互                |
|------|------|-------------------|
| **大纲/目录** | 树状结构管理卷/章/节，宏观把控 | 独立页面，未来要找一个设计感的方案 |
| **知识库-角色** | 角色卡片管理 | 独立页面，表单+列表        |
| **知识库-世界观** | 世界设定笔记 | 独立页面，分类+自由文本      |
| **知识库-功法** | 功法百科 | 独立页面，列表+详情        |
| **知识库-地点** | 地点/场景记录 | 独立页面，列表+详情        |
| **知识库-时间线** | 事件时间轴 | 独立页面，时间线视图        |

## 三、功能分层

### 核心层（MVP 必做）

#### A. 内容生成（聊天窗口内操作）

| 编号 | 功能 | 触发方式 | 反馈方式 | MVP 行为 |
|------|------|----------|----------|----------|
| A1 | 自由对话 | 直接输入 | AI 回复 | 与 AI 对话讨论写作思路 |
| A2 | 章节续写 | 选中文字 → 点续写 | AI 返回续写内容 | 直接返回 |
| A3 | 段落生成 | 输入指令 | AI 生成内容 | 直接返回 |
| A4 | 对话生成 | 输入指令 | AI 生成对白 | 直接返回 |
| A5 | 人物描写 | 输入指令 | AI 生成描写 | 直接返回 |

**架构设计**：
- 每个功能是独立的 Agent，有自己的 LangGraph 图和提示词
- API 按功能拆分，不搞一刀切
- 预留 Human-in-the-loop 节点（MVP 阶段跳过，后期打开）

##### A0 会话管理（聊天窗口基础设施）

| 编号 | 功能 | MVP 范围 | 远期扩展 |
|------|------|----------|----------|
| A0-1 | 历史对话保存 | ✅ 保存对话记录到 JSON | 支持多会话切换、搜索历史 |
| A0-2 | 对话延续 | ✅ 加载历史对话继续写 | 跨会话内容关联 |
| A0-3 | 对话导出 | ❌ 远期 | 导出为 .md/.txt/.docx |
| A0-4 | Token 用量看板 | ❌ 远期 | 每次会话的 Token/费用统计可视化 |

#### C. 结构与规划（独立页面）

| 编号 | 功能 | 说明 | MVP 范围 |
|------|------|------|----------|
| C1 | 小说大纲 | 整体故事大纲 | 树状结构编辑器，手动编辑 |
| C2 | 章节大纲 | 单章纲要 | 树状结构子节点编辑 |
| C3 | 情节弧线 | 起承转合 | 远期内置模板，MVP 自由文本 |
| C4 | 卷/篇结构 | 分卷规划 | 树状结构顶层节点 |

**数据模型**：卷 → 章 → 节，树形嵌套结构，保存时同步知识库。

**注意**：功法百科记录世界中的功法。角色对功法的掌握情况记录在角色卡中。

| 字段 | 类型 | 说明 |

##### D6 知识库检索

AI 写作时自动检索相关知识库内容并引用。MVP 阶段用简单的关键词/语义检索。架构预留向量检索切换入口。

##### D7 情节设计/设定灵感记录

独立笔记空间，记录未成型的情节创意、设定灵感。MVP 用自由文本 + 标签。

### 增强层（第二阶段）

#### B. 内容优化（聊天窗口内操作按钮）

| 编号 | 功能 | MVP 行为 | 架构预留 |
|------|------|----------|----------|
| B1 | 润色/打磨 | 直接返回结果 | LangGraph interrupt 节点（后续开启 HITL） |
| B2 | 扩写/缩写 | 直接返回结果 | 同上 |
| B3 | 风格转换 | 直接返回结果 | 同上，风格预设可配置 |
| B4 | 查错纠错 | 直接返回结果 | 同上 |
| B5 | 节奏分析 | 直接返回结果 | 同上 |

**架构设计**：
- B 类每个功能是独立的 Agent/Tool
- B 类 Agent 调用 E 类一致性检查 Tool 来验证结果
- MVP 阶段跳过 HITL 节点，直接返回；后续改造只需修改节点逻辑

#### E. 一致性与审查（底层共享能力，被 B 类调用）

| 编号 | 功能 | 说明 |
|------|------|------|
| E1 | 角色一致性检查 | 检查角色行为是否符合设定 |
| E2 | 世界观一致性检查 | 检查内容是否违背已设定规则 |
| E3 | 前后矛盾检测 | 同一/跨章节矛盾发现 |
| E4 | 伏笔管理 | 记录和追踪伏笔 |

**架构设计**：
- E 类是共享 Tool 层，被 B 类 Agent 在流程中调用
- 也可以被后台任务自动调用（写完一章后自动跑一遍）
- 每个 E 类工具输入为文本 + 相关知识库条目，输出为检查结果

### 模块分层职责

| 层 | 职责 | 调用方向 |
|------|------|----------|
| **api/** | HTTP 端点，参数校验，响应格式化 | → 调用 service 或 agent |
| **service/** | 业务逻辑，CRUD，事务 | → 调用 core/storage |
| **agents/** | AI 工作流编排，LangGraph 图 | → 调用 tools + core/llm |
| **tools/** | 可复用的原子能力（检索、检查等） | → 调用 core/storage |
| **core/** | 基础设施（LLM、存储、注册） | 无依赖 |
| **models/** | 数据结构定义（Pydantic） | 无依赖 |

### 关键数据流

**聊天 + 操作按钮**：
```
用户输入消息 → api_chat(agent="continue", message="选中的文字", ...)
              → get_agent("continue").ainvoke(state)
              → E 类工具检查一致性
              → LLM 生成
              → 直接返回（MVP）/ HITL 中断（远期）
```

**大纲编辑**：
```
前端操作 → api_outline CRUD → outline_service → storage.put("outlines", ...)
                                                    │
                                          写完后同步 ↓
                                     知识库文件自动更新
```

**知识库检索（D6）**：
```
Agent 写作中 → tools.retrieval.search("主角被退婚的场景")
              → core/storage 读取知识库文件
              → 返回相关角色/设定/时间线条目
              → Agent 拼入 LLM 上下文
```

## 六、扩展预留机制

| 扩展场景 | 改动范围 | 不改动 |
|---------|---------|--------|
| 新增 Agent | `agents/` 下新建目录 | 注册中心、API 路由、前端 |
| 新增 Tool | `tools/` 下新增文件 | 现有 Agent 代码 |
| 新增知识库实体 | `models/` 新增模型 + `services/` 新增 service | 其他实体 |
| B 类加 HITL | 修改对应 Agent 的 graph.py，打开 interrupt 节点 | API 接口、前端调用方式 |
| 换存储后端 | `core/storage.py` 新增实现类 | 所有 Service 和 Agent |
| 换 LLM | `core/llm.py` 加 provider 分支 | 所有 Agent |
| 前端换框架 | 重写 `frontend/` | 整个后端 |
| DX-1 角色虚拟形象 | 角色模型新增 `avatar_url` 等字段（已预留） | 角色 CRUD 逻辑 |
| DX-2 地点 3D 场景 | 地点模型新增坐标/模型字段（已预留） | 地点 CRUD 逻辑 |
