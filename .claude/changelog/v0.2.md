# v0.2 - 2026-06-25

## 变更概要

清理 v0.1 遗留问题，架构加固，完成产品设计文档，建立开发规范。

## 新增

### 产品文档
- `docs/产品设计文档.md` — 完整的功能分层、模块划分、数据流设计、扩展预留
- `docs/rule.md` — 文档命名规范（中文文件名、更新日期）
- `docs/后端代码说明.md` — 后端架构与请求流程文档

### 前端
- Next.js 16 + React 19 + shadcn/ui + Tailwind v4 项目初始化（`web/`）
- Bento Grid 宫格首页（6 个功能入口卡片）
- chat / dashboard / knowledge 子页面占位

### 项目规范
- `CLAUDE.md` 新增：
  - 变更联动规则（技术栈变更必须联动清理依赖和文档）
  - 启动脚本规则（关键文件变更不得破坏启动）
  - 前端技术栈声明（Next.js + shadcn/ui + Tailwind CSS）
- `start.bat` 一键启动脚本（自动清理端口、前后端并行启动）
- PyCharm Run Configurations（全栈/后端/前端各一套）

### 会话管理
- 产品文档新增 A0 会话管理模块（历史对话保存、延续、导出、Token 看板）

## 变更

### 架构重构
- **注册中心下沉**：`core/registry.py` 成为真正的组件注册中心（含发现逻辑），`agents/__init__.py` 退化为纯 re-export
- **路由分层**：`api/` 拆分为 `router.py`（主路由挂载）+ `routes/agents.py`（Agent 端点），支持未来按功能模块扩展

### 清理
- 移除 `pyproject.toml` 中残留的 `streamlit` 依赖（前端已切 Next.js）
- 删除根目录残留的 `main.py`
- 删除 `data/` 下空 `.gitkeep`

### 文档更新
- v0.1 changelog 中 Streamlit 描述已过时，v0.2 起改为 Next.js

## 技术选型

| 层面 | v0.1 | v0.2 |
|------|------|------|
| 前端 | Streamlit | **Next.js + shadcn/ui + Tailwind v4** |
| 注册中心 | 逻辑分散在 agents/__init__.py | **集中在 core/registry.py** |
| 路由 | 单文件 api/router.py | **分层 api/router.py + routes/** |
| 启动方式 | 手动分别启动 | **start.bat 一键启动 + PyCharm 配置** |
