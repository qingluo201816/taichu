## 当前功能范围

本版本能做：

- 导入小批量章节
- 编辑 Markdown 章节并保存
- Selection AI 生成 AIResultCard
- 将 suggestion 保存为 Idea
- 生成 PendingFact 候选
- 作者确认 PendingFact 成为 Knowledge
- Creative Inbox 管理非事实工作区资产
- 从 source 重建 generated SQLite/FTS 检索 projection
- 章节整理
- Basic Agent Chat
- Export readable JSON bundle

本版本不能做：

- 多小说管理
- 协作
- 发布平台
- 时间线 / 地图
- 自动知识入库
- 自动角色状态更新
- vector / embedding
- 复杂知识图谱
- Chat 历史会话持久化

## 核心边界

- 正文 Markdown 与 confirmed Knowledge JSON 是正式事实源。
- `project_assets/source/workspace/` 是工作区资产，不是默认 fact_scope。
- `project_assets/generated/` 可删除、可重建，不是事实源。
- SQLite 只能作为 generated projection，不能保存唯一用户资产。
- SourceRef 必须指向 source 中的 Chapter / Knowledge 等原始资产，不得指向 generated、SQLite row 或 `.db` 文件。
- AI 输出必须是 AIResultCard。
- PendingFact 未经作者确认前不是事实，不得进入 Knowledge 或默认 fact_scope。

## RC1 已验证项

- 自动化 smoke 已覆盖 Chat SourceRef 不指向 `project_assets/generated`、不包含 `/sqlite/`、不以 `.db` 结尾。
- 自动化 smoke 已覆盖 Chat citation label 从 `S1` 开始，与来源顺序保持一致。
- 自动化 smoke 已覆盖 Export bundle 包含 workspace `pending_facts.jsonl`，且不导出 `generated/` 路径。
- 浏览器 smoke 已覆盖 `/chat` 提交消息、显示 AIResultCard 和 source path。
- 浏览器 smoke 已覆盖 `/settings` 执行 generated rebuild、导出 bundle 并显示 completed job。

## 冻结规则

RC1 之后不再继续扩功能。只允许：

- bug fix
- 文档
- release note
- 验收脚本
- E2E 修复
- 启动 / 部署修复
- 真实数据试用中发现的阻塞问题

禁止：

- 新功能
- 新页面
- 新 Agent
- 新数据模型
- 新工作流
- 新知识结构
- vector / embedding
- 复杂检索
- 自动入库
- 多小说 `project_id` / `novel_id`

## 后续人工验收

正式 `v0.1.0` 发布前，还需要完成：

1. 使用真实小说 3-5 章进行 dogfooding。
2. 验证导入、编辑保存、刷新不丢失。
3. 验证 Selection AI 的 ask / enrich_setting / continue_text。
4. 验证 Idea 保存、PendingFact 转入 Inbox、确认 Knowledge。
5. 验证 generated rebuild 后 Chat 能基于 confirmed Knowledge 返回 SourceRef。
6. 验证 Export bundle 包含 `source/`，不包含 `generated/`。
7. 在安全时机关闭当前服务后运行 `start.bat`，确认前后端可正常启动。

若无阻塞 bug，可从 `v0.1.0-rc1` 晋级为 `v0.1.0`；若有阻塞 bug，只做验收修复并发布下一个 RC。
