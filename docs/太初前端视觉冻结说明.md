# 太初前端视觉冻结说明

> 更新日期：2026-06-30

## 设计来源

本视觉基线以根目录 `TAICHU_DESIGN.md` 为强制规则源，适用于 `web/` 下页面、组件、样式、交互、动效和用户可见文案。旧版设计说明已归档到 `docs/太初旧版前端设计已废弃.md`，只用于历史追溯，不再作为当前实现依据。

## 当前视觉基线

太初前端统一为“奶油纸张画布 + 深墨绿锚点 + 薰衣草高亮 + 编辑出版感字体”的编辑科技风格：

- 奶油纸张画布用于主要页面、功能入口、资料阅读和浅色内容区。
- 深墨色用于正文、边框、深色展示区和高对比结构。
- 深墨绿用于品牌锚点、资料状态、强调带和重要归档信息。
- 薰衣草色用于每屏唯一主操作、标题高亮、选区提示和轻量状态。
- 展示标题使用 EB Garamond 与中文衬线 fallback；功能文字使用 Figtree 与中文无衬线 fallback。
- 所有用户可见文案必须为中文，不能直接展示英文内部字段名、接口名或枚举值。

## Token 迁移状态

当前 `web/src/app/globals.css` 中仍可能保留旧版兼容 token。后续前端迁移时必须以根目录 `TAICHU_DESIGN.md` 中的新 token 为准：

- `--tc-cream-paper`
- `--tc-midnight-ink`
- `--tc-stone-mist`
- `--tc-graphite-veil`
- `--tc-smoke`
- `--tc-deep-forest-teal`
- `--tc-lavender-whisper`
- `--tc-amber-pulse`
- `--tc-font-display`
- `--tc-font-ui`
- `--tc-font-mono`

旧 token 可作为迁移期兼容层保留，但新增组件、新页面、新样式和自动前端任务不得继续扩展旧视觉方向。

## 页面覆盖范围

- `/`：应作为太初写作助手的清晰入口，第一屏能识别品牌、单本玄幻小说写作场景和主要进入动作。
- `/home`：功能入口使用奶油纸张画布、精致卡片和明确中文模块名。
- `/editor`：章节编辑、工具栏、资料辅助和 AI 操作遵循编辑出版工具语言，优先保障输入效率。
- `/inbox`：灵感、待确认资料和 AI 内容片段以可归档编辑材料呈现。
- `/knowledge`：知识列表和详情以资料档案感呈现，来源、状态和章节关系清晰可见。
- `/chat`：对话表单、结果正文和来源证据使用统一卡片与徽标体系。
- `/settings`：导出、重建、清理等低频操作保持清晰、克制、中文说明完整。
- `/dashboard`：如果保留，应与当前视觉基线一致，不单独创建新的后台风格。

## 组件覆盖范围

- `Button`、`Card`、导航、输入框、标签、弹层和状态提示应逐步迁移到当前 token。
- 主按钮使用薰衣草底、深墨色文字、深墨色 1px 边框，且每屏只保留一个。
- 卡片默认使用奶油纸或白色表面、32px 圆角和细边框。
- 标签和徽标必须配中文文字，不得只靠颜色表达状态。
- AI 结果和来源证据必须显示中文操作与中文来源说明。

## 禁止随意修改的边界

- 不引入游戏 UI、赛博霓虹、普通后台管理系统或多色高饱和按钮体系。
- 不把薰衣草色铺成大面积页面背景。
- 不把纯黑作为常规正文或默认页面背景。
- 不隐藏 AI 卡片、PendingFact、Knowledge、来源证据的关键操作。
- 不把英文内部字段名直接作为用户可见文案。
- 不复制外部参考产品的营销文案、平台下载语义、客户 logo 或业务内容。

## 允许未来修复范围

后续视觉变更应限制为：

- 与根目录 `TAICHU_DESIGN.md` 一致的 token 迁移。
- 样式 bug 修复。
- 响应式适配。
- 可访问性改进。
- 性能优化。
- 组件库准入后的局部实现补齐。

不得在没有新设计确认的情况下更换整体视觉方向。
