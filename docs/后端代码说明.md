# 后端代码说明

> 更新日期：2026-06-24

## 概述

Taichu 后端基于 **FastAPI + LangGraph + LangChain**，提供 RESTful API 供前端调用。核心设计遵循**可扩展性优先**原则——新增 Agent 只需新建目录，不改已有代码。

## 目录结构

```
src/taichu/
├── main.py                 # FastAPI 入口，CORS 配置
├── config.py               # 配置管理（.env 驱动）
├── core/
│   ├── llm.py              # LLM 工厂（按 provider 创建实例）
│   ├── storage.py          # 存储抽象层（接口 + JSON 实现）
│   └── registry.py         # Agent 注册中心（薄封装）
├── agents/                 # Agent 插件目录
│   ├── __init__.py          # Agent 自动发现引擎
│   ├── base.py             # Agent 协议定义
│   └── chat/               # Chat Agent（示例）
│       ├── graph.py        # LangGraph 图定义
│       ├── nodes.py        # 图节点实现
│       └── prompts.py      # 系统提示词
├── api/
│   ├── deps.py             # FastAPI 依赖注入
│   └── router.py           # 路由定义
└── models/
    └── schemas.py          # Pydantic 请求/响应模型
```

## 请求流程

一次对话请求的完整链路：

```
POST /api/chat  {"agent": "chat", "message": "你好"}
  │
  ▼
api/router.py          → 解析请求，获取 agent 名称
  │
  ▼
core/registry.py       → 查找 agent
  │
  ▼
agents/__init__.py     → 自动发现 + 调用 build_graph() 获取编译后的图
  │
  ▼
agents/chat/nodes.py   → call_model() 构造消息（系统提示词 + 用户消息）
  │
  ▼
core/llm.py            → get_llm() 返回 DeepSeek LLM 实例
  │
  ▼
DeepSeek API           → 远程调用，返回 AI 回复
  │
  ▼
HTTP 200  {"agent": "chat", "response": "..."}
```

## 各模块说明

### config.py

基于 pydantic-settings，从 `.env` 文件和环境变量读取配置。包含：
- LLM 相关：provider、api_key、api_base、model
- 服务端：host、port
- 存储：data_dir（JSON 文件存储目录）

### core/llm.py — LLM 工厂

提供统一的 LLM 创建入口 `get_llm(provider=None)`。目前支持 DeepSeek，扩展新模型只需在函数内加一个分支。DeepSeek 使用 OpenAI 兼容接口，通过 `ChatOpenAI` 封装。

### core/storage.py — 存储抽象

定义 `StorageBackend` 抽象基类（get/list/put/delete），`JsonStorageBackend` 是其 JSON 文件实现。每个 collection 对应 `data/` 下的子目录，每个 key 对应一个 `.json` 文件。未来替换为 SQLite 或其他数据库，只需新增一个实现类。

### agents/ — Agent 自动发现

这是扩展性的核心。`__init__.py` 在启动时扫描 `agents/` 下所有子目录，寻找包含 `graph.py` 的目录，自动导入并注册。

**每个 Agent 目录约定**：
- 必须包含 `graph.py`
- `graph.py` 必须导出两个符号：
  - `agent_meta`：`{"name", "label", "description"}` 元信息字典
  - `build_graph()`：返回编译后的 LangGraph 图
- `prompts.py` 和 `nodes.py` 为可选，但推荐分离

**新增 Agent 示例**：在 `agents/` 下新建目录（如 `agents/outline/`），创建 `graph.py` 导出 `agent_meta` 和 `build_graph()`，无需修改任何已有代码即可被自动发现。

### api/router.py — 路由

- `GET /api/agents`：列出所有已注册的 Agent
- `POST /api/chat`：统一对话入口，根据 `agent` 参数路由到对应 Agent 图执行

### models/schemas.py — 数据模型

- `ChatRequest`：`agent`（模块名）+ `message`（用户输入）
- `ChatResponse`：`agent` + `response`（AI 回复）
- `AgentInfo`：`name` + `label` + `description`

## 扩展指南

| 扩展场景 | 操作 |
|---------|------|
| 新增 Agent | `agents/` 下新建目录，实现 `graph.py` |
| 新增 LLM 提供商 | `core/llm.py` 的 `get_llm()` 加分支 |
| 替换存储后端 | 实现 `StorageBackend` 子类，修改 `deps.py` 注入 |
| 新增 API 端点 | `api/router.py` 添加路由 |
