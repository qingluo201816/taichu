# 前端风格切换任务

> 更新日期：2026-06-30

## 任务背景

太初当前前端设计规则源是根目录 `TAICHU_DESIGN.md`，当前主风格为“奶油纸张画布 + 深墨绿锚点 + 薰衣草高亮 + 编辑出版感字体”的 Wispr Flow 参考方向。

未来希望支持多套完整视觉风格的一键切换。这里的切换不是简单白天/黑夜模式，也不是局部换色，而是“全站设计皮肤切换”：所有页面、组件、按钮、卡片、编辑器、资料页和 AI 状态在同一套布局与功能结构下，整体切换到另一种参考 design 的视觉语言。

## 已确认选择

用户已确认以下边界：

- 允许每套风格修改颜色。
- 允许每套风格修改字体。
- 允许每套风格修改圆角。
- 允许每套风格修改边框。
- 允许每套风格修改阴影。
- 允许每套风格修改局部动效强度。
- 不允许每套风格修改页面信息结构。
- 不允许每套风格修改组件功能。

换句话说，风格包可以改变“看起来像什么”，但不能改变“页面有什么、功能怎么工作、信息如何组织”。

## 推荐方案

推荐采用“语义 token + 风格包切换”方案。

组件不直接使用某个具体风格名的 token，而是使用稳定的语义 token，例如：

```css
--tc-surface-page
--tc-surface-card
--tc-surface-panel
--tc-text-primary
--tc-text-secondary
--tc-text-muted
--tc-border-subtle
--tc-border-strong
--tc-action-primary-bg
--tc-action-primary-text
--tc-accent-brand
--tc-accent-highlight
--tc-radius-control
--tc-radius-card
--tc-shadow-card
--tc-font-display
--tc-font-ui
--tc-motion-fast
--tc-motion-normal
```

每一种风格只负责给这些语义 token 赋值。组件只关心“页面背景”“卡片背景”“主按钮”“弱边框”“正文颜色”等语义，不关心当前风格参考的是哪一个 design。

## 技术方向

建议未来实现时使用 HTML 属性控制风格：

```html
<html data-theme-style="wispr-editorial">
```

切换到另一套风格时改为：

```html
<html data-theme-style="midnight-editorial">
```

主题选择使用 `localStorage` 持久化，首次进入时使用默认风格。除非后续有明确必要，不建议新增主题依赖。

建议文件位置：

```text
web/src/components/theme/
  theme-provider.tsx
  theme-switcher.tsx
  theme-registry.ts

web/src/app/globals.css
  基础语义 token
  [data-theme-style="wispr-editorial"] 风格赋值
  [data-theme-style="..."] 其他风格赋值
```

## 实施顺序建议

1. 先在 `TAICHU_DESIGN.md` 增加“多风格切换”章节，明确风格包边界、语义 token 和禁止项。
2. 在 `globals.css` 建立语义 token 层，把当前 Wispr 风格映射成第一套风格包。
3. 逐步把组件里的具体 token 引用迁移到语义 token。
4. 增加本地 `ThemeProvider` 和全站风格切换控件。
5. 增加第二套参考 design 风格包，验证多风格机制是否稳定。
6. 手动检查主要页面：入口、首页、编辑器、收件箱、知识库、对话、设置页。

## 重要约束

- 所有用户可见文案必须保持中文。
- 不复制外部参考产品的营销文案、客户 logo、下载平台语义或业务内容。
- 每套风格必须有明确 design reference，不允许随意拼贴。
- 布局结构和功能行为保持一致。
- 风格切换不能破坏编辑器输入、AI 卡片操作、来源证据、知识库阅读和设置页低频操作。
- 新增风格不得引入游戏 UI、赛博霓虹、普通后台管理系统或高饱和多色按钮体系，除非未来明确修改项目视觉边界。

## 验收标准

- 用户可以在界面中一键切换风格。
- 刷新页面后保留上次选择的风格。
- 所有主要页面同步切换，没有明显残留旧风格的页面或组件。
- 同一套风格内颜色、字体、圆角、边框、阴影和局部动效保持一致。
- 页面信息结构、路由、表单、AI 操作和数据行为不因风格切换改变。
- 文本对比度满足长时间阅读，尤其是编辑器正文、资料卡、来源证据和按钮文字。
