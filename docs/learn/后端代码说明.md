# 后端代码说明

> 更新日期：2026-06-25

## 概述

太初后端基于 FastAPI、LangGraph 和 LangChain，服务当前唯一的一本玄幻小说。代码采用 `api → application → domain` 分层，存储、检索、LLM、MCP 和插件发现由 `infrastructure` 实现。

完整架构与未来功能落点以 [太初项目架构](../../.claude/management/PROJECT_ARCHITECTURE.md) 为准。

## 启动组装

`main.py` 是唯一组合入口：

1. 从 `config.py` 取得 settings。
2. 创建 JSON Storage 和配置指定的 LLM。
3. 将能力放入 `CapabilityContext`。
4. 使用 `plugin_discovery.py` 发现 Agent 和 Tool 候选。
5. 交给应用层 Registry 校验并注册。
6. 将 Registry 和 Storage 放入 FastAPI 应用状态。
7. 挂载 API 路由。

模块不得自行创建 LLM、存储或注册中心。

## Agent 请求链路

```text
POST /api/chat
→ api/routes/agents.py
→ AgentRegistry.get_graph()
→ application/agents/chat/graph.py
→ 注入的 LLM
→ 返回 ChatResponse
```

`GET /api/agents` 返回 Agent Manifest 的前端可见字段：

- `name`
- `label`
- `description`
- `required_capabilities`
- `exposures`
- `supports_streaming`

## Agent 插件

Agent 协议定义在 `application/agents/contract.py`。每个 Agent 子目录的 `graph.py` 导出：

- `manifest: AgentManifest`
- `build_graph(context: CapabilityContext) -> CompiledStateGraph`

新增 Agent 只需创建 `application/agents/{name}/` 并实现协议，不修改发现器、注册中心或固定 Agent 列表。

插件发现与注册严格分离：

- `infrastructure/plugin_discovery.py`：扫描和动态导入。
- `application/agents/registry.py`：能力校验、重名检查、注册和查询。

## 存储与数据态

`StorageBackend` 使用 `Protocol` 定义在 `application/contracts/storage.py`。当前 JSON 实现在 `infrastructure/storage/json_backend.py`，提供异步 CRUD，并阻止路径穿越。

```text
project_assets/
├── source/          # 当前唯一小说的事实来源
└── generated/       # 可删除、可重建的索引和缓存
```

`config.py` 使用 `PROJECT_ASSETS_DIR` 指定数据态根目录，默认值为 `project_assets`。

## 当前 API

| 方法 | 路径 | 用途 |
|---|---|---|
| GET | `/api/agents` | 查询已注册 Agent |
| POST | `/api/chat` | 调用指定 Agent |
