# 太初前端设计说明

> 更新日期：2026-06-27
> 强制适用范围：太初前端视觉系统、页面布局、组件设计、交互动效、Tailwind token、shadcn/ui 二次封装和所有用户可见前端文案。

本文档是太初前端设计的强制规则源。涉及 `web/` 下页面、组件、样式、交互、动效、前端文案或视觉方案的创建、修改、评审和讨论时，必须先读取并遵循本文档。该要求同样适用于任务包执行、自动前端开发、自动修复、自动重构和前端 review，不以用户是否显式提出“使用设计规则”或“使用组件库”为前提。不得使用其他视觉规则覆盖本文档，也不得将本文档仅作为可选参考。

## 1. Design Philosophy：设计哲学

太初是面向单本玄幻小说的个人 AI 写作助手。它不是普通 SaaS，不是游戏 UI，也不是单一风格页面。太初前端必须按写作流程分成三层视觉系统：

```text
Outer layer: Aaru-inspired dark particle observatory void.
Workspace layer: Langbase-inspired refined dark monochrome writing console.
Document layer: Switch-Lit-inspired cream-paper editorial cards.
```

中文定义：

```text
外层是太初虚空。
内层是冷静写作工作台。
内容卡片像一张张从黑暗中浮出的纸质稿件。
```

分层原因：

- 入口页承担“进入世界”的仪式感。它应像玄幻宇宙的观测站，而不是产品营销首页。
- 主工作台承担长时间写作。它必须克制、低对比、稳定，避免装饰打断作者注意力。
- AI 生成内容承担“沉淀为资料”的感受。设定、人物、伏笔、摘要、导出预览应像纸质稿件，从深色工作台中浮现。
- AI 状态是辅助读数，不是彩色主角。状态提示要像小型仪表、检索读数、生成脉冲，而不是大面积按钮或光效。

三份参考的吸收方式：

- Aaru 只影响入口页与 AI 状态仪表语言。
- Langbase 是主应用工作台外壳的主要参考。
- Switch-Lit 只影响内容卡片、稿件预览和导出页面。

## 2. Visual Layers：视觉层

### 2.1 Entry Layer：入口层

入口层是“太初虚空”。它使用 Aaru 的深黑观测站气质，但服务于玄幻写作场景。

视觉关键词：

- 纯黑画布。
- 暖白或白色粒子。
- 点云地景。
- 远景最高位面宫阙剪影。
- 极简白框 `TAICHU` 品牌按钮，附中文动作文案。
- 少量坐标、阶段、世界种子、AI 激活读数。
- 酸黄绿色小型状态灯，只用于“世界种子已点亮”“AI 已激活”等状态。

入口页布局应保持极简：

- 页面全屏，无普通导航栏。
- 中心偏下或视觉焦点处放置 `TAICHU` 白框入口。
- 粒子不要均匀随机散布，应形成点云地景、位面层次或观测场。
- 远景宫阙只用低透明度线稿或剪影，不能变成插画主角。
- 状态读数只能少量出现，例如“世界种子：已激活”“坐标：太初虚空”“阶段：观测中”。

入口层禁止：

- 禁止米纸背景。
- 禁止普通 SaaS landing page。
- 禁止随机星空。
- 禁止霓虹赛博。
- 禁止游戏 HUD。
- 禁止复杂功能 UI。

### 2.2 Workspace Layer：工作台层

工作台层是“冷静写作控制台”。它吸收 Langbase 的深灰控制台、细线网格、终端精度和极弱极光，但必须围绕写作效率设计。

视觉关键词：

- 深色灰阶画布，不使用纯黑作为主工作区背景。
- 左侧章节树。
- 中间章节编辑器。
- 右侧 AI / 资料工作台。
- 顶部工具栏与底部状态栏。
- 1px 细边框。
- 低对比灰色层级。
- 紧凑布局。
- 单一克制极光作为 Selection AI 或生成状态提示。

工作台应优先保护写作沉浸：

- 编辑器是中心，其他面板不能抢占视觉权重。
- 所有操作按钮默认使用灰阶、白色或透明边框。
- AI 的存在应表现为“弱极光、细线、读数、状态点”，不是大面积彩色入口。
- 面板之间用 hairline border 和灰阶层级区分，不用强阴影。

工作台禁止：

- 禁止大面积米纸底。
- 禁止高饱和装饰。
- 禁止杂志式大标题压过编辑器。
- 禁止游戏化按钮、技能槽、能量条。
- 禁止强光效。
- 禁止到处漂浮的卡片。

### 2.3 Paper-card Layer：纸面内容层

纸面内容层吸收 Switch-Lit 的暖米纸、墨线、编辑排版和荧光标签。它只用于“内容沉淀”，不是主应用外壳。

适用内容：

- AI 生成的设定卡。
- 人物卡。
- 地点卡。
- 势力卡。
- 伏笔卡。
- 章节摘要卡。
- 资料确认卡。
- 导出预览。
- 阅读稿件和打印稿感页面。

视觉关键词：

- 暖米纸背景。
- 黑色 1px 墨线边框。
- 强标题层级。
- 稿件分栏和编辑网格。
- 小面积荧光绿、春绿或浅蓝标签。
- 纸张从深色工作台中浮出的感觉。

纸面内容层禁止：

- 禁止把整个主应用变成米纸色。
- 禁止覆盖入口页。
- 禁止把所有按钮做成杂志风。
- 禁止让高亮色大面积铺底。

### 2.4 AI-state / Indicator Layer：AI 状态与仪表层

AI 状态层横跨入口和工作台，但始终保持小面积、仪表化、读数化。

适用内容：

- 世界种子状态。
- 模型连接状态。
- AI 生成中。
- 检索中。
- Selection AI 可用。
- 来源引用数量。
- 知识库同步状态。

表现方式：

- 小圆点、小短线、细进度条、低透明读数。
- 酸黄绿色只用于入口和世界种子级别状态。
- 极光只用于主工作台中的 AI 激活、Selection AI、生成状态。
- 状态文案必须是中文，例如“生成中”“检索完成”“来源 3 条”。

## 3. Color Tokens：颜色 Token

所有 token 使用 CSS variables。全局可声明，但使用范围必须由组件层限制。

### 3.1 Void Tokens：入口虚空

```css
:root {
  --tc-void-black: #000000;
  --tc-void-near: #050505;
  --tc-void-carbon: #0a0a0a;
  --tc-void-particle: #f7f3ea;
  --tc-void-particle-muted: #b8b2a8;
  --tc-void-frame: #ffffff;
  --tc-void-frame-warm: #f4efe5;
  --tc-void-label: #9d9d9d;
  --tc-void-label-dim: #6f6f6f;
  --tc-void-seed: #ebfb10;
  --tc-void-seed-soft: rgba(235, 251, 16, 0.16);
  --tc-void-palace-line: rgba(244, 239, 229, 0.28);
}
```

使用规则：

- `--tc-void-black` 只用于入口页和极少数过渡背景。
- `--tc-void-seed` 只做小点、小线、状态灯，不做大按钮。
- 粒子以暖白为主，灰色为辅，不使用彩色星空。

### 3.2 Workspace Tokens：主工作台灰阶

```css
:root {
  --tc-workspace-bg: #151516;
  --tc-workspace-shell: #0f0f10;
  --tc-workspace-panel: #1c1c1e;
  --tc-workspace-panel-soft: #202023;
  --tc-workspace-recess: #101012;
  --tc-workspace-editor: #181819;
  --tc-workspace-border: #3a3a3f;
  --tc-workspace-border-weak: #29292d;
  --tc-workspace-grid: rgba(255, 255, 255, 0.035);
  --tc-workspace-text: #f4f1ea;
  --tc-workspace-text-secondary: #b7b3aa;
  --tc-workspace-text-muted: #85858b;
  --tc-workspace-text-dim: #5f5f66;
  --tc-workspace-focus: #fafafa;
}
```

使用规则：

- 主应用 `body` 或工作区容器使用 `--tc-workspace-bg`，不要使用入口的纯黑。
- 面板使用 `--tc-workspace-panel`，输入区使用 `--tc-workspace-recess`。
- 层级靠 1px 边框、灰阶差异和留白建立，不靠彩色块。

### 3.3 Aurora Tokens：AI 激活与生成状态

```css
:root {
  --tc-aurora-peach: #f6d1ac;
  --tc-aurora-rose: #f3b5d2;
  --tc-aurora-violet: #c7b8f5;
  --tc-aurora-mint: #a7eadc;
  --tc-aurora-sky: #afcdf6;
  --tc-aurora-line: rgba(199, 184, 245, 0.48);
  --tc-aurora-soft: rgba(167, 234, 220, 0.08);
  --tc-aurora-glow: rgba(167, 234, 220, 0.16);
  --tc-aurora-gradient: linear-gradient(
    90deg,
    #f6d1ac 0%,
    #f3b5d2 24%,
    #c7b8f5 48%,
    #a7eadc 72%,
    #afcdf6 100%
  );
}
```

使用规则：

- 极光只用于 Selection AI、生成中状态、AI 结果边缘提示和极少量背景扫描线。
- 极光不能作为普通按钮背景。
- 极光不能作为正文文字颜色。
- 极光不能覆盖编辑器正文。
- 单屏内极光面积应小于可视面积的 3%。

### 3.4 Paper Tokens：纸质稿件

```css
:root {
  --tc-paper-bg: #f8f3e6;
  --tc-paper-bg-soft: #fbf7ee;
  --tc-paper-ink: #11100d;
  --tc-paper-ink-muted: #6f675c;
  --tc-paper-border: #11100d;
  --tc-paper-border-soft: #d8cdbb;
  --tc-paper-shadow: rgba(0, 0, 0, 0.18);
  --tc-paper-lime: #edfe5e;
  --tc-paper-spring: #31e992;
  --tc-paper-cornflower: #bed4fb;
  --tc-paper-mark-soft: rgba(237, 254, 94, 0.36);
}
```

使用规则：

- `--tc-paper-bg` 只能用于内容卡片、稿件预览、导出页内的纸张区域。
- `--tc-paper-ink` 用于纸面标题、正文、边框和线性图标。
- 荧光色只用于标签、批注标记、状态短条和小 swatch。
- 纸面卡片可以投下极轻阴影，但不能像普通 SaaS 卡片漂浮。

## 4. Typography：字体系统

### 4.1 UI 字体

主 UI 使用几何无衬线：

```css
--tc-font-ui: "Geist", "Inter", "PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif;
```

适用范围：

- 导航。
- 面板标题。
- 按钮。
- 工具栏。
- 状态栏。
- 表单。
- 命令面板。
- AI 状态文本。

规则：

- 工作台 UI 使用 12px、13px、14px、15px 为主。
- 字重以 400、500 为主，谨慎使用 600。
- 小标签可使用轻微正字距或正常字距，但不要影响中文可读性。

### 4.2 Mono 字体

技术读数和仪表文本使用等宽字体：

```css
--tc-font-mono: "Geist Mono", "JetBrains Mono", "SFMono-Regular", Consolas, monospace;
```

适用范围：

- 坐标读数。
- token、id、引用编号。
- 模型状态。
- 检索状态。
- 时间戳。
- 命令面板快捷键。

规则：

- 等宽字体不能用于长段中文正文。
- 等宽读数必须配中文解释，例如“来源 3 条”，不要只显示 `source_ref_count: 3`。

### 4.3 编辑器正文字体

章节编辑器以长时间中文写作为核心，应提供安静、可读的正文体验：

```css
--tc-font-editor: "Noto Serif SC", "Source Han Serif SC", "Songti SC", "SimSun", serif;
```

建议：

- 正文默认 17px 到 18px。
- 行高 1.75 到 1.9。
- 段落最大行宽 38 到 46 个中文字符。
- 编辑器背景仍是暗色，不切换为米纸。
- 标题可用 UI 字体或编辑器字体，但不能变成杂志海报式大标题。

### 4.4 内容卡片标题和正文

纸面内容层可以使用衬线字体增强稿件感：

```css
--tc-font-paper-title: "Noto Serif SC", "Source Han Serif SC", "Songti SC", serif;
--tc-font-paper-body: "Noto Serif SC", "Source Han Serif SC", "Songti SC", serif;
```

适用范围：

- 人物卡标题。
- 设定卡标题。
- 章节摘要标题。
- 导出预览标题。
- 阅读稿件正文。

纸面元信息仍使用 UI 字体或等宽字体：

- 标签。
- 来源徽标。
- 状态。
- 卡片操作按钮。

### 4.5 Serif 使用边界

可以使用 serif 的位置：

- 编辑器正文。
- 阅读模式正文。
- 纸面内容卡标题。
- 纸面内容卡正文。
- 导出预览。

不能使用 serif 的位置：

- 主导航。
- 左侧章节树。
- 右侧 AI 工具面板。
- 工具栏按钮。
- 命令面板。
- Toast。
- 状态读数。
- Selection AI 操作菜单。

## 5. Layout Rules：布局规则

### 5.1 入口页布局

入口页为全屏沉浸入口。

结构：

```text
EntryViewport
  ParticleFieldCanvas
  DistantPalaceLayer
  VoidReadoutLayer
  TaichuEntryAction
```

规则：

- `ParticleFieldCanvas` 占满视口，使用点、短线、极弱网格或地形粒子。
- `DistantPalaceLayer` 位于远景上三分之一或中远景，透明度低，不能抢占入口按钮。
- `VoidReadoutLayer` 放置 2 到 5 个小型状态读数。
- `TaichuEntryAction` 是唯一主要交互，白框、透明底、中文辅助说明。
- 首屏不得出现产品功能卡片、定价、营销段落。

### 5.2 主工作台三栏布局

桌面端默认三栏：

```text
WorkspaceShell
  TopToolbar        高 44-52px
  MainGrid
    ChapterTree     宽 260-304px
    EditorPane      minmax(560px, 1fr)
    AiDataPane      宽 320-400px
  StatusBar         高 24-32px
```

规则：

- 左侧章节树：章节、卷、状态、字数，以紧凑列表呈现。
- 中间编辑器：永远是视觉中心，背景低对比，正文行宽受控。
- 右侧 AI / 资料面板：包含 Selection AI、知识条目、资料确认、引用来源。
- 三栏之间使用 1px `--tc-workspace-border-weak`。
- TopToolbar 不做大导航，只放当前章节、保存状态、模型状态、少量工具。
- StatusBar 放字数、保存、检索、生成状态。

响应式：

- 小于 1100px：右侧 AI 面板折叠为抽屉。
- 小于 820px：左侧章节树折叠为侧边抽屉，中间编辑器优先。
- 移动端不强行三栏，优先保证章节编辑和返回入口。

### 5.3 Selection AI 浮层

Selection AI 是选中文本后的轻量操作层。

规则：

- 浮层贴近选区，避免遮挡正在编辑的句子。
- 宽度建议 280-360px。
- 背景使用 `--tc-workspace-panel`。
- 边框使用 `--tc-workspace-border`。
- 顶部或左侧可以出现 1px 极光线。
- 操作项使用中文动词，例如“润色”“续写”“提炼设定”“生成伏笔”。
- 禁止使用大面积渐变底。
- 禁止使用弹窗式大卡片打断写作。

### 5.4 内容卡片布局

纸面卡片只出现在内容沉淀场景。

结构：

```text
PaperCard
  CardHeader
    Title
    TypeTag
  CardBody
  Evidence / SourceRef
  CardActions
```

规则：

- 默认宽度 320-520px，详情页可扩展到 720px。
- 背景使用 `--tc-paper-bg`。
- 边框使用 1px `--tc-paper-border`。
- 圆角不超过 8px。
- 标题使用 serif，正文使用 serif，元信息使用 UI 字体。
- 标签使用小面积 `--tc-paper-lime` 或 `--tc-paper-spring`。
- 卡片进入工作台时应和深色背景形成明确层次，但不允许铺满整个工作台。

### 5.5 导出 / 预览页布局

导出预览页是“黑暗中的纸张”。

规则：

- 外层仍是深色工作台背景。
- 中间显示单页或双页米纸稿件。
- 工具栏保持暗色，放在顶部或右侧。
- 页面内容使用纸面 token。
- 打印边距、章节标题、页码、注释、引用以稿件排版方式呈现。
- 不要把导出页做成普通 PDF 工具，也不要把工具按钮做成纸面杂志按钮。

## 6. Component Rules：组件规则

### 6.1 Button：按钮

按钮按层分三类。

入口按钮：

- 透明背景。
- 1px 暖白边框。
- `TAICHU` 作为品牌字样，旁边或下方提供中文动作说明“进入太初”。
- 方角或 2px 小圆角。
- hover 只提高边框亮度和粒子响应。

工作台按钮：

- 默认灰阶。
- 主操作可使用白底黑字 pill，但只能在当前上下文有明确主动作时使用。
- 次级操作使用透明底、1px 灰边框。
- 图标按钮优先使用 lucide 图标，并提供中文 tooltip。
- 不使用高饱和色按钮。

纸面按钮：

- 只用于纸面卡片内部。
- 米纸底、黑色 1px 边框。
- 小面积荧光底只用于确认类标签或重点标记，不做全局 CTA。

### 6.2 Panel：面板

工作台面板：

- 背景 `--tc-workspace-panel` 或 `--tc-workspace-panel-soft`。
- 1px 边框。
- 圆角 0 到 8px。
- 不使用厚阴影。
- 面板嵌套时内层不要再做完整卡片外观，改用分隔线和标题行。

纸面面板：

- 仅用于纸质稿件预览和内容卡片。
- 不作为主应用导航或主工作台容器。

### 6.3 Editor：编辑器

编辑器是主工作台中心。

规则：

- 背景使用 `--tc-workspace-editor`。
- 正文使用编辑器字体，字色 `--tc-workspace-text`。
- 选区使用低透明极光或暖白，不使用系统默认高饱和蓝。
- 当前段落可以用左侧细线或极弱背景提示。
- 保存中、AI 处理中等状态进入底部状态栏，不覆盖正文。
- 编辑器内禁止出现大面积纸面卡片，AI 结果应在右侧或浮层出现。

### 6.4 AI Result Card：AI 结果卡

AI 结果卡根据结果类型决定外观。

临时结果：

- 留在右侧 AI 面板内。
- 使用深色面板样式。
- 顶部可有极光细线。
- 操作包括“插入正文”“转为设定卡”“丢弃”。

已沉淀结果：

- 转为纸面卡片。
- 使用 `PaperCard`。
- 标注类型、来源、更新时间。
- 内容必须可追溯到章节或资料来源。

### 6.5 Knowledge Card：知识卡

知识卡用于世界观、人物、地点、势力、伏笔等资料。

列表态：

- 在右侧工作台内可以使用深色紧凑行。
- 展示名称、类型、来源数量、更新时间。

详情态：

- 使用纸面卡片。
- 标题使用稿件标题样式。
- 正文分区展示“事实”“推断”“待确认”“关联章节”。
- 不直接暴露内部字段名，例如 `source_refs` 应显示为“来源”。

### 6.6 SourceRef / citation badge：来源徽标

来源徽标用于说明资料来自哪里。

规则：

- 文案使用“来源”“章节”“选区”“手动确认”等中文。
- 可以用等宽字体显示短编号，例如 `#012`。
- 在深色面板中使用灰阶边框徽标。
- 在纸面卡片中使用黑色细线徽标。
- 点击后打开来源预览，不跳出主工作流。

### 6.7 Tag：标签

标签用于类型、状态、阶段。

工作台标签：

- 深色底或透明底。
- 灰色边框。
- 字号 11-12px。
- 中文短词，例如“草稿”“待确认”“已入库”。

纸面标签：

- 使用 `--tc-paper-lime`、`--tc-paper-spring` 或 `--tc-paper-cornflower` 小面积填充。
- 黑色文字和黑色细边框。
- 不使用整行荧光色背景。

### 6.8 Command Palette：命令面板

命令面板属于工作台层。

规则：

- 深色浮层。
- 宽度 560-720px。
- 顶部输入框使用 recessed surface。
- 结果列表紧凑，左侧为命令名，右侧为快捷键或范围。
- 命令名必须中文，例如“打开章节”“搜索设定”“生成章节摘要”。
- 快捷键可使用等宽字体。
- 不使用纸面风格。

### 6.9 Modal：模态框

默认模态框属于工作台层。

规则：

- 背景遮罩使用深色半透明。
- 模态框使用深色面板。
- 需要展示稿件、设定卡、导出页时，模态内部可以嵌入纸面内容区。
- 不要把模态框整体改成米纸，除非它是纯稿件预览。
- 破坏性操作必须使用中文说明和明确按钮，不用红色大面积警示。

### 6.10 Toast / Status Indicator：提示与状态

Toast 和状态提示必须克制。

规则：

- 默认出现在右下角或底部状态栏。
- 深色背景、1px 边框、短中文文案。
- 成功、失败、生成中不依赖高饱和色块。
- 世界种子和 AI 激活可用小型酸黄绿色点。
- Selection AI 生成中可用极光短线动画。
- 错误状态使用文本和图标说明，不使用大面积红色。

## 7. Motion Rules：动效规则

### 7.1 入口页点云进入动画

规则：

- 进入时粒子从低透明度逐步显现，时长 900-1400ms。
- 点云可以从地平线或中心场缓慢聚合。
- 宫阙线稿延迟 200-400ms 出现。
- `TAICHU` 白框按钮在粒子稳定后淡入。
- 动画应像观测数据建立，不像星空闪烁。

禁止：

- 禁止随机闪烁星星。
- 禁止高速飞线。
- 禁止镜头冲刺。
- 禁止强粒子爆炸。

### 7.2 AI 生成状态动画

规则：

- 使用小型状态点、1px 扫描线、极弱极光条。
- 循环周期 1200-1800ms。
- 生成中文案保持稳定，例如“生成中”“检索中”“整理来源”。
- 不使用整屏 loading。

### 7.3 Selection AI 出现 / 消失

规则：

- 出现：透明度 0 到 1，位移 4-8px，时长 120-160ms。
- 消失：透明度 1 到 0，时长 90-120ms。
- 不做弹跳。
- 不改变编辑器布局。

### 7.4 Paper Card 浮现动画

规则：

- 纸面卡片从深色面板中淡入。
- 位移 6-12px。
- 时长 180-260ms。
- 可带极轻纸面阴影，但不能强悬浮。
- 多张卡片依次出现时，间隔不超过 60ms。

### 7.5 禁止过度动画

全局规则：

- 尊重 `prefers-reduced-motion`。
- 不使用持续旋转装饰。
- 不使用强 glow。
- 不使用大面积渐变流动背景。
- 不让动效干扰输入、选中文本和阅读。

## 8. Usage Boundaries：使用边界

明确边界是太初设计系统的核心。

### 8.1 Aaru 的边界

Aaru 只能影响：

- 入口页。
- 点云入口。
- `TAICHU` 白框按钮。
- 世界种子状态灯。
- AI 激活小读数。
- 少量坐标、阶段、状态读数。

Aaru 不能影响：

- 主工作台大面积背景。
- 章节编辑器。
- 内容卡片。
- 普通按钮体系。
- 导出预览纸面。

### 8.2 Langbase 的边界

Langbase 是主工作台外壳。

适用：

- 三栏工作台。
- 面板。
- 工具栏。
- 状态栏。
- 命令面板。
- 输入框。
- 检索状态。
- Selection AI 浮层。

不适用：

- 入口页的纯黑虚空气质。
- 纸面内容卡的米纸和墨线。
- 稿件导出页内部纸张。

### 8.3 Switch-Lit 的边界

Switch-Lit 只用于内容卡片和稿件预览。

适用：

- 设定卡。
- 人物卡。
- 地点卡。
- 势力卡。
- 伏笔卡。
- 章节摘要卡。
- 资料确认卡。
- 导出预览。
- 阅读稿件。

不适用：

- 入口页。
- 主工作台背景。
- 左侧章节树。
- 工具栏。
- 命令面板。
- Selection AI 操作菜单外壳。

### 8.4 不允许无边界混用

禁止以下混用：

- 在入口页放米纸卡片。
- 在主工作台铺满米纸背景。
- 在纸面卡片里使用大面积极光渐变。
- 用酸黄绿色做全局主按钮。
- 把所有状态都做成粒子和坐标读数。
- 把编辑器做成杂志封面。

## 9. Implementation Notes：实现说明

### 9.1 技术栈

前端实现基于：

- Next.js。
- Tailwind CSS。
- shadcn/ui。
- lucide-react 图标。

所有面向用户的文案必须为中文。内部组件名、token 名、文件路径可以使用英文，但不能把英文内部标识直接显示给用户。

涉及前端组件库、花哨组件、视觉素材、动画组件或 registry 组件选择时，必须同时使用项目级 Skill：`.agents/skills/taichu-ui-components/SKILL.md`。该 Skill 是自动前端任务的组件准入规则，不限于用户显式要求。

### 9.2 CSS variables 与 Tailwind token 组织

建议在 `web/src/app/globals.css` 或 `web/src/styles/tokens.css` 中声明全局 token：

```css
:root {
  /* void */
  --tc-void-black: #000000;
  --tc-void-seed: #ebfb10;

  /* workspace */
  --tc-workspace-bg: #151516;
  --tc-workspace-panel: #1c1c1e;
  --tc-workspace-border: #3a3a3f;
  --tc-workspace-text: #f4f1ea;

  /* aurora */
  --tc-aurora-gradient: linear-gradient(90deg, #f6d1ac, #f3b5d2, #c7b8f5, #a7eadc, #afcdf6);

  /* paper */
  --tc-paper-bg: #f8f3e6;
  --tc-paper-ink: #11100d;
  --tc-paper-lime: #edfe5e;
}
```

Tailwind 侧映射建议：

```ts
// tailwind.config.ts 示例片段
theme: {
  extend: {
    colors: {
      "tc-workspace": "var(--tc-workspace-bg)",
      "tc-workspace-panel": "var(--tc-workspace-panel)",
      "tc-workspace-border": "var(--tc-workspace-border)",
      "tc-paper": "var(--tc-paper-bg)",
      "tc-paper-ink": "var(--tc-paper-ink)",
      "tc-void": "var(--tc-void-black)",
      "tc-seed": "var(--tc-void-seed)"
    }
  }
}
```

shadcn/ui 建议：

- shadcn 默认主题变量映射到 workspace 层，不映射到 paper 层。
- `background`、`foreground`、`border` 默认使用工作台灰阶。
- 纸面组件通过局部 class 或组件级 CSS variables 覆盖。
- 不要让 `body` 背景变成 `--tc-paper-bg`。

### 9.3 组件目录建议

建议按视觉边界分目录：

```text

  distant-palace.tsx
  void-readout.tsx
  taichu-entry-button.tsx

web/src/components/workspace/

  chapter-tree.tsx

  status-bar.tsx

web/src/components/ai-state/
  ai-indicator.tsx

web/src/components/paper-card/

  ai-result-card.tsx
  knowledge-card.tsx
  source-ref-badge.tsx

```

目录边界：

- `entry/` 不引用 `paper-card/`。
- `workspace/` 可以引用 `ai-state/`。
- `workspace/` 只在内容沉淀区域引用 `paper-card/`。
- `paper-card/` 不依赖入口粒子或虚空读数。

### 9.4 全局 token 与局部样式

应进入全局的 token：

- 四组颜色 token。
- 字体 token。
- 常用半径 token。
- 常用边框 token。
- 常用 z-index token。

只在局部组件启用的样式：

- 入口页粒子动画。
- 远景宫阙线稿。
- 极光渐变扫描线。
- 纸面卡片的米纸背景和墨线。
- 导出页纸张尺寸。
- Selection AI 的定位和出现动画。

### 9.5 可访问性与交互

规则：

- 所有按钮和图标按钮必须有中文可访问名称。
- 纸面荧光标签不能只靠颜色表达状态，必须配文字。
- 深色工作台文本对比度必须满足长时间阅读。
- 编辑器、命令面板、Selection AI 必须支持键盘操作。
- `prefers-reduced-motion` 下关闭粒子聚合、极光流动和卡片浮现位移。

### 9.6 组件库准入

太初允许使用更丰富的组件素材，但只能作为局部实现材料，不能成为新的全局视觉系统。

优先级：

- 优先复用 `web/src/components/ui/` 和现有业务组件。
- 基础交互优先使用 shadcn/ui、Base UI 和 lucide-react。
- 动效类素材可参考 Motion Primitives 一类的基础动效组件，但必须保持克制。
- 视觉类素材可参考 Magic UI 一类的局部粒子、光线、扫描线或状态动效，但必须改造成太初入口层、工作台层、纸面层或 AI 状态层语言。
- Aceternity UI 一类组件只能作为局部源码素材或视觉参考，不得直接套用营销区块、霓虹背景、炫光卡片或完整 landing page。

自动任务规则：

- 自动前端任务默认也要执行组件库准入，不需要用户显式提出。
- 引入外部 registry 或新增依赖前，必须先判断本地组件是否足够。
- 外部组件落地后必须使用太初 token、中文文案和本项目目录结构。
- 不允许把外部组件原始品牌色、英文示例文案或默认营销布局留在界面里。

## 10. Anti-goals：明确禁止项

太初前端不追求以下方向：

- 游戏 UI。
- 赛博霓虹。
- 大面积米纸主应用。
- 普通 SaaS landing page。
- 随机星空粒子。
- 过度炫光。
- 高饱和按钮。
- 无边界混搭。
- 入口页功能堆叠。
- 工作台装饰压过编辑器。
- AI 状态变成大型彩色组件。
- 把英文内部标识直接作为用户可见文案。

最终判断标准：

```text
进入太初时，用户看到的是黑暗中的观测站。
开始写作时，用户进入的是冷静、低干扰的写作工作台。
内容沉淀时，用户看到的是从黑暗中浮出的纸质稿件。
```
