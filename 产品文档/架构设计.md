## 权威来源

代码目录、分层职责、依赖方向、插件协议、数据态结构和未来功能落点统一以 [太初项目架构](../.claude/management/PROJECT_ARCHITECTURE.md) 为权威来源。

本文只说明产品级数据流和扩展预留，不重复维护代码目录树，避免产品文档与开发规范形成两套架构标准。

## 产品边界

- 太初只服务当前唯一的一本玄幻小说。
- 不提供小说列表、小说切换、跨小说检索或 `project_id`。
- 用户提到“主角”“世界观”“当前章节”等内容时，默认属于当前唯一小说。
- 小说内部的人物、地点、势力和事件仍使用稳定 ID 关联。

## 前后端交互

```text
Next.js 页面
→ web/src/lib/api/
→ FastAPI api/routes/
→ application/services/ 或 application/agents/
→ application/contracts/
→ infrastructure/
→ project_assets/source/ 与 generated/
```

- 普通知识库、大纲和正文操作由 Service 承载。
- AI 写作和审查流程由 Agent 编排。
- Agent 可调用应用 Tool，但不直接访问文件、向量库、LLM Provider 或 MCP。
- 前端只通过 FastAPI API 使用后端能力。

## 关键数据流

### 聊天与写作操作

```text
用户输入或选中文本
→ Agent API
→ AgentRegistry 获取对应 Agent
→ LangGraph 工作流
→ 检索 Tool 获取当前小说相关设定
→ LLM 生成
→ 直接返回（MVP）或进入 HITL（后续）
```

### 大纲编辑

```text
前端编辑卷/章/节
→ Outline Service
→ StorageBackend
→ project_assets/source/plots/outlines/
→ Index Service 更新 generated/ 中的可重建索引
```

源数据只保存一次。知识检索索引是派生数据，不再通过“大纲文件 + 知识库文件”双写形成两个事实来源。

### 知识库检索

```text
Agent 或审查 Tool
→ RetrievalBackend.search()
→ 关键词、向量或混合检索实现
→ project_assets/source/ + generated/search_index|vector_store
→ 返回统一 RetrievalResult
```

检索范围固定为当前唯一小说，不需要小说选择参数。

### 插件启动注册

```text
main.py
→ infrastructure/plugin_discovery.py 发现候选
→ application/agents/registry.py 或 tools/registry.py 校验注册
→ API、UI 或 MCP 按 exposures 暴露能力
```

插件发现与协议注册是独立职责，不得合并。

## 扩展预留

| 扩展场景 | 主要落点 | 不应修改 |
|---|---|---|
| 新增 Agent | `application/agents/{name}/` | 发现器、注册中心、固定 Agent 列表 |
| 新增 Tool | `application/tools/` | 现有 Agent 的内部实现 |
| 新增知识库实体 | `domain/models/` + `application/services/` | 其他领域实体 |
| 开启 HITL | 对应 Agent Graph | 领域模型和存储实现 |
| 更换存储 | `infrastructure/storage/` | Service、Agent 和领域层 |
| 更换检索方案 | `infrastructure/retrieval/`、`indexing/` | 调用检索契约的用例 |
| 新增 LLM Provider | `infrastructure/llm/providers/` | Agent 工作流 |
| 调用外部 MCP | `infrastructure/mcp/` | 领域层 |
| 新增独立产品页面 | `web/src/app/{feature}/` | 通用 Agent 注册机制 |
| DX-1 角色形象 | 角色领域模型、媒体基础设施、角色页面 | 其他知识库实体 |
| DX-2 地点 3D 场景 | 地点领域模型、场景基础设施、地点页面 | 其他知识库实体 |
