# 项目制作人的话
目前的初步大方向是左侧是大纲,顶部是编辑器的功能,右侧是ai赋能,选中文字也能进行ai或业务逻辑赋能
## 定位

写作页面是太初的**核心创作空间**，将大纲结构管理与正文写作合并在同一视图中，取代早期产品设计中"大纲独立页面 + 聊天窗口写作"的割裂模式。

用户在左侧管理卷/章/节的树形结构，在中间区域专注写作，在右侧调用 AI 能力辅助创作。

### 三栏说明

| 区域 | 默认宽度 | 行为 |
|------|---------|------|

| 编辑器 | 自适应填充 | 核心区域，永远可见 |

### 交互

| 操作 | 触发方式 | 行为 |
|------|---------|------|
| 切换章节 | 单击章节点 | 编辑器加载该章内容 |

### 状态标记

每个章节点支持状态：

| 状态 | 图标 | 说明 |
|------|------|------|
| 草稿 | ○ | 尚未开始写或仅有关键词 |
| 写作中 | ◐ | 已有内容但未完成 |
| 已完成 | ● | 本章内容完成 |
| 需修改 | ◉ | 标记需要回头修改 |

状态切换：右键菜单 → 设置状态，或节点上点击状态图标循环切换。

## 编辑器

### 核心体验

参考小黑屋、橙瓜等网文写作工具的通用设计：

- **打字机滚动**：编辑区域始终将当前书写行保持在视口垂直居中，避免写到底部需手动滚动
- **自动保存**：停止输入 2 秒后自动保存，状态栏显示"上次保存：X 秒前"
- **专注模式**：点击全屏或快捷键（F11），隐藏大纲树和 AI 面板，仅保留编辑器 + 底部状态栏
- **暗色/护眼/亮色主题**：编辑器独立于系统主题，支持三种配色切换

### 格式支持

MVP 阶段支持基础 Markdown 渲染：

| 格式 | 语法 | 渲染效果 |
|------|------|---------|
| 章节标题 | `# 标题` / `## 小节` | 对应字号、间距 |
| 加粗 | `**文字**` | 加粗 |
| 斜体 | `*文字*` | 斜体 |
| 分割线 | `---` / `***` | 居中的 `※ ※ ※` 或横线 |
| 引用 | `> 引用内容` | 左侧竖线缩进 |

原因：网文正文格式需求简单（主要是标题、强调、场景切换分割），Markdown 即可覆盖，无需富文本的复杂度。

### 快捷键

| 快捷键 | 操作 |
|--------|------|
| Ctrl+S | 手动保存 |
| Ctrl+B | 加粗 |
| Ctrl+I | 斜体 |
| F11 | 专注模式 |
| Ctrl+[ | 折叠大纲树 |
| Ctrl+] | 展开 AI 面板 |
| Tab | 首行缩进（写入两个全角空格 `　　`） |

## AI 操作

### 两种触发方式

**1. 浮动工具栏（选中文字后弹出）**

```
┌──────────────────────┐
│ 选中文字区域           │
│                      │
│    "主角缓缓抬手..."   │
│                      │
│       ↓              │
│ ┌──────────────────┐ │
│ │ 📝续写  🎨润色  📏扩写│ │
│ │ 📏缩写  🔄风格  🔍查错│ │
│ └──────────────────┘ │
└──────────────────────┘
```

行为：
- 鼠标划选超过 10 个字后，在选区上方浮现工具栏
- 点击操作按钮将选中文字作为上下文发送给对应 Agent
- 支持快捷键触发（如选中后 Ctrl+Enter 直接续写）

**2. 右侧 AI 操作面板**

固定展示所有 `exposures` 包含 `"ui"` 的 Agent，按类别分组：

```
AI 操作面板
───────────────
📝 写作辅助
  章节续写
  段落生成
  对话生成
  人物描写

🎨 内容优化
  润色打磨
  扩写/缩写
  风格转换
  查错纠错

📊 分析工具
  节奏分析
  搜设定
  检查一致性

💡 灵感
  相关灵感生成
```

行为：
- 点击 Agent 按钮，面板内展开输入框或直接将当前编辑器选中内容作为输入
- Agent 生成结果在编辑器内以差异视图（建议模式）或直接插入形式呈现
- 可以一键接受/拒绝 AI 建议

### AI 结果反馈方式

| 模式 | 行为 | 适用场景 |
|------|------|---------|
| 追加 | AI 内容直接追加到光标位置 | 续写、灵感生成 |
| 替换 | 替换选中文字 | 润色、风格转换 |
| 建议 | 编辑器内高亮显示 AI 修改建议，用户逐条接受/拒绝 | 查错纠错、一致性审查 |

## 数据模型

### 大纲节点（Outline Node）

```python
class OutlineNode:
    id: str              # 稳定唯一 ID，如 "ch_001"
    parent_id: str | None # 父节点 ID，卷节点为空
    type: str            # "volume" | "chapter"
    title: str           # 展示标题，如 "第一章：星火初燃"
    sort_order: int      # 同层级排序
    status: str          # "draft" | "writing" | "completed" | "revision"
    word_count: int      # 字数（从正文内容计算，不手动填写）
    target_words: int    # 目标字数
    created_at: str      # ISO 时间
    updated_at: str      # ISO 时间
```

### 正文内容（Chapter Content）

```python
class ChapterContent:
    node_id: str         # 关联 OutlineNode.id
    content: str         # Markdown 正文
    version: int         # 版本号，每次保存递增
    updated_at: str
```

正文以 Markdown 文件存储在 `project_assets/source/manuscripts/chapters/{node_id}.md`，与大纲节点分离存储。大纲树是索引，正文是内容。

## 核心交互流程

### 新建章节并开始写作

```
用户点击"新增章"
→ 大纲树末尾创建草稿节点，进入重命名
→ 用户输入标题，回车确认
→ 编辑器创建空文档，光标定位第一行
→ 状态设为"写作中"
→ 用户开始输入，计时器启动自动保存
```

### AI 辅助写作

```
用户在编辑器中选中一段文字
→ 浮动工具栏弹出
→ 点击"续写"
→ 系统将选中的文字 + 当前章节上下文发送给 continuation Agent
→ Agent 生成内容
→ 以追加模式插入光标位置（可 Ctrl+Z 撤销）
→ 字数统计自动更新
```

### 切换章节

```
用户点击大纲树中的"第4章"
→ 自动保存当前正在编辑的内容
→ 编辑器加载第4章正文
→ 状态栏更新为第4章信息
→ 面包屑更新
```

## 与 Agent 的集成点

| Agent | 触发场景 | 输入 | 输出位置 |
|-------|---------|------|---------|
| continuation | 选中文字 + 续写 | 选中文字 + 上下文 | 光标位置追加 |
| generation | AI 面板点击 | 用户指令 | 光标位置追加 |
| polishing | 选中文字 + 润色 | 选中文字 | 替换选区 |
| rewriting | 选中文字 + 扩写/缩写 | 选中文字 | 替换选区 |
| style_transfer | 选中文字 + 风格转换 | 选中文字 | 替换选区 |
| proofreading | 选中文字 + 查错 | 选中文字 | 建议模式叠加 |
| pacing | AI 面板点击 | 当前章节 | 面板内显示分析结果 |
| review | AI 面板点击 | 当前章节 | 建议模式叠加 |
