# 产品基线：太初 MVP v1

## 1. 产品定位

太初是面向单个作者、单本玄幻长篇小说的个人 AI 创作 IDE。它以当前章节编辑器为主战场，把章节正文、小说设定、角色状态、世界观资料、AI 工作流、灵感沉淀、设定检索和冲突检查压缩进一个连续写作环境。

太初不是：

- 通用写作软件
- 网文发布平台
- 多人协作 SaaS
- 多小说项目管理平台
- 一键自动写书工具
- 纯 Agent 技术演示平台

## 2. 第一用户

第一用户是项目作者本人：混合型、偏边写边发现的中文玄幻长篇作者。系统要服务“边写正文、边发现设定、边让 AI 补充、边确认沉淀”的写法。

## 3. 测试语料边界

开发阶段可使用《太初》（校对版全本）作者高楼大厦作为测试语料，用于章节切分、摘要、设定抽取、角色状态追踪、全文索引、向量检索和长篇上下文召回测试。

该测试语料必须与原创小说事实源物理隔离。开发态可以切换测试数据目录；产品态不提供多小说管理。

## 4. MVP v1 定义

MVP v1：编辑器 + AI 卡片 + 创作收件箱/轻看板 + 最小知识库 + 混合检索 + 防 AI 污染事实闸门。

第一版验证闭环：

```text
进入当前章节
→ 正常写作
→ 选中文本 + 简单提示词 / 设定补充 / 续写 XX 字
→ AI 以卡片形式给出结果
→ 作者选择插入正文、保存为灵感、生成待确认设定、或丢弃
→ 系统生成本章摘要草稿和待确认设定变更
→ 作者确认后才进入正式知识库
→ 下一次 AI 检索只使用正文和已确认知识，并能给出来源依据
```

## 5. 第一版三个高频 AI 工作流

### 5.1 选区问答

用户选中正文后输入问题，例如：

- 这段的问题是什么？
- 这里能不能更有压迫感？
- 这个角色动机合理吗？
- 我想表达天命感，怎么改？

输出：SuggestionCard；如果使用检索证据，必须带 SourceRef。不得直接修改正文。

### 5.2 选区设定补充

用户选中秘境、功法、宗门、法宝、角色背景等正文，让 AI 补充设定建议。

输出：SuggestionCard 或 PendingFactCard。不得直接写 Knowledge。

### 5.3 选区续写 XX 字

用户选中上下文，指定 200 / 500 / 1000 / 自定义字数。

输出：TextCandidateCard。默认只输出正文，不解释、不跳剧情、不擅自新增重大设定。若产生可能新增设定，只生成 PendingFact，不直接写 Knowledge。

## 6. 北极星验收

一次真实写作中，作者能在不离开章节编辑器的情况下：

1. 写作并保存当前章节。
2. 使用三类选区 AI。
3. 插入至少一条 AI 续写正文。
4. 保存至少一条 AI 建议为灵感。
5. 生成至少一条 PendingFact。
6. 确认 PendingFact 后写入 Knowledge JSON。
7. 下一次检索能引用该知识来源。
8. 未确认设定、灵感、未采纳正文不会被当成正式事实。
9. 正文和知识库可导出。
10. 删除 generated 后可重建索引。
