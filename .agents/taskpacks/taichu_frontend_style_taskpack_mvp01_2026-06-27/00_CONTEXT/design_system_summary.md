# 三层设计系统摘要

> 更新日期：2026-06-27

本摘要只用于帮助 Codex 快速理解方向；具体 token 和细节以 `TAICHU_DESIGN.md` 为准。

## 第一层：入口虚空

气质：玄幻、浩瀚、静谧、克制、高级。

用途：进入太初之前的仪式感。它可以是点云虚空、最高位面、世界种子等视觉语言，但不能抢走工作台的使用效率。

前端改造要求：若入口页已存在，本任务只检查它和整体设计系统的衔接，不重写入口大场景。入口层的色彩、字号、按钮、跳转应与工作台层一致。

## 第二层：深色写作工作台

气质：暗色低饱和、边界克制、长期写作不刺眼、信息密度高但不拥挤。

用途：承载主应用框架：导航、章节侧栏、AI 侧栏、卡片区域、收件箱、知识库、设置、对话页。

关键要求：不要做赛博朋克，不要做强霓虹，不要做游戏 HUD，不要喧宾夺主。主工作区应该安静、稳定、可信。

## 第三层：纸质稿件内容层

气质：纸感、温润、可读、适合中文长文。

用途：正文编辑区、正文候选卡、长文本阅读区、章节摘要、知识详情中的大段文本。

关键要求：正文不能是纯黑背景上的高对比白字；编辑器应该像“在暗色工作台上放了一张可长期书写的稿纸”。

## 层级关系

入口虚空负责“进入感”；深色工作台负责“掌控感”；纸质稿件负责“写作感”。

三者不能混淆：不要把入口页的强视觉粒子带入正文编辑器；不要把纸质稿件色铺满整个工作台；不要把工作台做成普通 SaaS 灰白后台。
