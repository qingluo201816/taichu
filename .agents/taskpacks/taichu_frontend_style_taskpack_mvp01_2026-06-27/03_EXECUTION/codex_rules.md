# Codex 执行规则

> 更新日期：2026-06-27

## 总规则

1. 每次只执行一个 Phase。
2. Phase 0 未完成并获得用户批准前，不得改样式代码。
3. 不得修改后端和 `project_assets`。
4. 不得改变 API、数据结构、业务状态机。
5. 所有视觉改造必须可回溯到 `TAICHU_DESIGN.md` 或 `STYLE_LOCK.md`。
6. 不得用大面积硬编码代替 token。
7. 不得把入口虚空视觉带进主工作台。
8. 不得降低中文正文可读性。
9. 不得隐藏 AI 卡片、PendingFact、Knowledge、Evidence 的关键状态。
10. 不得在无法判断路由和组件职责时自行猜测。

## 工作方式

每个 Phase 开始时：

1. 重读本 Phase 文件。
2. 检查 `STYLE_LOCK.md`。
3. 输出简短执行计划。
4. 列出预计修改文件。
5. 明确不会修改的范围。

每个 Phase 结束时：

1. 运行可用检查。
2. 按证据模板返回。
3. 停止，等待用户指令进入下一 Phase。

## 文档规则

`docs/` 下新增文档必须：

- 文件名用中文，除 `rule.md` 本身。
- 包含更新日期。

## 包管理规则

先读取当前 lockfile 和 `web/package.json`，遵守仓库现有 package manager。不要无故新增依赖。样式改造原则上不需要新增重型依赖。
