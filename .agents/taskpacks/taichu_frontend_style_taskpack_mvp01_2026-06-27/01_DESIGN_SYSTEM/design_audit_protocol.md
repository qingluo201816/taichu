# 设计审计协议

> 更新日期：2026-06-27

Phase 0 必须输出 `STYLE_AUDIT.md`。审计范围：

## 页面审计

逐页列出：

- 路由
- 页面用途
- 当前视觉层级
- 应映射到的设计层
- 与 `TAICHU_DESIGN.md` 的差距
- 是否纳入本次改造

## 组件审计

逐组件列出：

- 组件路径
- 业务用途
- 当前样式来源
- 是否硬编码颜色/尺寸
- 是否应抽 token
- 风险等级

## Token 审计

列出：

- 当前全局 CSS 变量
- Tailwind 使用方式
- 硬编码颜色
- 重复阴影/边框/半径
- 与三层设计系统冲突的样式

## 风险审计

必须指出：

- 哪些样式改动可能破坏 Tiptap 编辑器。
- 哪些样式改动可能破坏 AI 卡片操作。
- 哪些样式改动可能影响响应式布局。
- 哪些样式改动可能影响可访问性。

## 输出要求

`STYLE_AUDIT.md` 只记录发现，不直接改代码。
