# 三层设计系统页面映射

> 更新日期：2026-06-27

Codex 必须先扫描实际路由，再在 `STYLE_LOCK.md` 中写清每个页面属于哪一层。

## 推荐映射

| 页面/组件 | 所属层 | 改造重点 |
|---|---|---|
| `/entry` | 入口虚空 | 保持仪式感，与工作台色彩衔接，不重写大场景 |
| `/` 当前工作台 | 深色写作工作台 | 应成为稳定应用入口或工作台壳 |
| `/editor` | 深色工作台 + 纸质稿件 | 三栏结构，中心稿纸，右侧 AI 卡片 |
| 章节导航 | 深色工作台 | 低对比、层级清晰、当前章节明显 |
| 正文编辑器 | 纸质稿件 | 中文长文可读性、纸感、行宽、段距 |
| AI 卡片面板 | 深色工作台 | 卡片层级、动作按钮、状态明确 |
| 正文候选卡 | 纸质稿件 | 候选正文应接近稿纸质感 |
| 建议/灵感卡 | 深色工作台 | 信息简洁，状态可扫读 |
| PendingFact 卡 | 深色工作台 | 待确认状态必须显眼但不刺眼 |
| Evidence 卡 | 深色工作台 | 来源依据可识别、可跳转 |
| `/inbox` | 深色工作台 | 轻看板，不做项目管理系统 |
| `/knowledge` | 深色工作台 + 纸质详情 | 列表暗色，详情可用纸质阅读区 |
| `/chat` | 深色工作台 | 不抢主入口，输出可卡片化 |
| `/settings` | 深色工作台 | 简洁、低频、不主导视觉 |

## 关键边界

- 入口虚空的粒子、宫阙、强转场不要进入编辑器正文区。
- 纸质稿件层不要铺满所有页面。
- 工作台层不要变成灰白 SaaS 后台。
