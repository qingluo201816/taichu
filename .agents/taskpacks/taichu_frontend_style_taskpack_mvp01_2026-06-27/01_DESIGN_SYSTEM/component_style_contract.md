# 组件样式契约

> 更新日期：2026-06-27

目标：让 MVP0.1 关键组件统一成太初视觉语言，但不改业务语义。

## App Shell

应具备：

- 暗色背景。
- 左侧导航/章节区与中间内容区分层明确。
- 顶部状态条或轻量标题区低调存在。
- 当前工作上下文清楚：当前章节、保存状态、当前 AI 工作流。

不得：

- 大面积纯黑无层级。
- 大面积高饱和霓虹。
- 像普通后台管理系统。

## Editor

应具备：

- 中心纸质稿件层。
- 合理中文行宽。
- 段落呼吸感。
- 选区工具与 AI 操作不打扰正文。
- 保存状态清晰但不喧宾夺主。

不得：

- 纯黑编辑区。
- 代码编辑器感。
- 过多边框包围正文。

## AI Cards

应具备：

- 明确卡片类型：正文候选、建议、待确认设定、证据、章节整理、灵感。
- 明确操作区：插入、替换、保存、确认、忽略、丢弃、重试。
- 状态色克制。
- 来源证据可见。

不得：

- 卡片像普通聊天气泡。
- 重要操作隐藏到 hover 才能发现。
- 待确认设定看起来像已确认事实。

## Inbox

应具备：

- 灵感、待确认设定、AI 收藏、待处理项分区明确。
- 轻量看板，不做复杂项目管理。
- 可以快速扫读和回到来源。

## Knowledge

应具备：

- 知识卡列表清晰。
- 详情区适合阅读长设定。
- 来源、状态、别名、类型可识别。
- confirmed / archived / pending 不混淆。

## Buttons and Inputs

- 主按钮少而明确。
- 危险动作需克制但明确。
- 输入框适合中文写作，不要过小。
- focus-visible 必须可见。

## Empty / Loading / Error

所有页面都必须有符合太初风格的空态、加载态、错误态。不要出现默认白底错误块。
