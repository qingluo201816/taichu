# Phase 6：Visual QA / Freeze

> 更新日期：2026-06-27

## 阶段目标

完成视觉验收，冻结 MVP0.1 后前端视觉基线，形成后续不再随意修改的说明文档。

## 要实现什么

1. 根据验收清单逐页检查。
2. 修复明显视觉不一致。
3. 生成或更新 `docs/太初前端视觉冻结说明.md`。
4. 文档必须包含更新日期，符合 `docs/rule.md`。
5. 总结已覆盖页面、未覆盖页面、后续只允许修复的范围。

## 不要实现什么

- 不再引入新的大风格。
- 不再重构业务结构。
- 不再新增功能。

## 文档建议内容

`docs/太初前端视觉冻结说明.md`：

- 更新日期
- 设计来源：`TAICHU_DESIGN.md`
- 三层设计系统摘要
- token 实现位置
- 页面覆盖范围
- 组件覆盖范围
- 禁止随意修改的视觉边界
- 允许未来修复范围：bug、性能、适配、可访问性
- 验收结果

## 验收标准

- `npm run lint` 通过。
- `npm run build` 通过。
- 如有 `npm run test:editor`，尽量运行并记录结果。
- 视觉冻结说明存在。
- 用户能按手动验收步骤检查主页面。

## 测试要求

- 页面手动 walkthrough。
- 编辑器输入与保存不受影响。
- AI 卡片操作不受影响。
- 收件箱/知识库/对话页可访问。
- docs 命名符合规则。

## 主要风险

- 最后一阶段又引入新的风格方向。
- 文档没有约束力，后续继续乱改。

## 给 Codex 的任务拆解

1. 运行完整检查。
2. 修复小问题。
3. 生成冻结文档。
4. 返回最终证据。
