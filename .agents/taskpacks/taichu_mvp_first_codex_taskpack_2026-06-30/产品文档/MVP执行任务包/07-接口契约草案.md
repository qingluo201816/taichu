# 07-接口契约草案

> 更新日期：2026-06-30

## 1. 大纲与章节

```text
GET    /api/outline
POST   /api/outline/volumes
PATCH  /api/outline/volumes/{volume_id}
POST   /api/outline/chapters
PATCH  /api/outline/chapters/{chapter_id}
GET    /api/chapters/{chapter_id}
PUT    /api/chapters/{chapter_id}
```

`GET /api/outline` 响应示例：

```json
{
  "volumes": [
    {
      "volume_id": "volume-001",
      "name": "第一卷 大田初醒",
      "order": 1,
      "chapters": [
        {
          "chapter_id": "chapter-001",
          "display_title": "第1章 大田金鳞元神出",
          "order": 1
        }
      ]
    }
  ],
  "current_volume_id": "volume-001",
  "current_chapter_id": "chapter-001"
}
```

## 2. 知识库

```text
GET    /api/knowledge/types
GET    /api/knowledge/cards?type=character&status=all&q=
POST   /api/knowledge/cards
GET    /api/knowledge/cards/{id}
PATCH  /api/knowledge/cards/{id}
POST   /api/knowledge/cards/{id}/mark-active
POST   /api/knowledge/cards/{id}/mark-deprecated
```

状态筛选：全部、草稿、有效；废弃通过“只看废弃”单独查看。前端不得直接显示英文枚举。

## 3. Inbox

```text
GET    /api/inbox?tab=ideas
POST   /api/inbox/ideas
PATCH  /api/inbox/ideas/{id}
GET    /api/inbox/pending-facts
POST   /api/inbox/pending-facts
PATCH  /api/inbox/pending-facts/{id}
POST   /api/inbox/pending-facts/{id}/confirm
GET    /api/inbox/issues
POST   /api/inbox/issues
PATCH  /api/inbox/issues/{id}
```

待确认事实确认入库时，必须选择知识卡类型，并传入最终知识卡预览。确认成功后，原 Inbox 记录保留，状态变为已处理。

## 4. 写作区 AI 对话

必须使用：

```text
POST /api/ai-workspace-conversations
POST /api/ai-workspace-conversations/{id}/messages
```

辅助接口：

```text
GET /api/ai-workspace-conversations
GET /api/ai-workspace-conversations/{id}
```

创建对话示例：

```json
{
  "chapter_id": "chapter-001",
  "task_type": "continue",
  "subtask_type": null,
  "reference_scope": "chapter",
  "model_name": "mock-llm"
}
```

发送消息示例：

```json
{
  "user_input": "续写 200 字，压迫感更强",
  "reference": {
    "scope": "chapter",
    "chapter_id": "chapter-001",
    "selected_text": "",
    "chapter_text": "本章正文参考内容"
  }
}
```

发送消息接口返回完整对话。

## 5. AI 历史

```text
GET /api/ai-history?chapter_id=&task_type=&has_source=&has_error=
GET /api/ai-history/{conversation_id}
```

筛选：章节、功能入口、是否有来源、是否有错误。模拟输出只作为标记显示，不作为筛选条件。

## 6. 设置

```text
GET   /api/settings/preferences
PATCH /api/settings/preferences
```

第一版只返回基础偏好，不返回真实 LLM 配置。

## 7. 错误响应

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "名称、摘要和来源引用补齐后，才能标记为有效。"
  }
}
```

前端只能展示中文 `message`，不得展示英文 `code`。
