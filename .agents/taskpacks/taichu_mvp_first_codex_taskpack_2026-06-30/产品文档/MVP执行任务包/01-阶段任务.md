## Phase 1：数据结构

目标：建立 MVP 第一版数据结构和存储契约。

允许修改：`src/taichu/domain/**`、`src/taichu/application/**`、`src/taichu/infrastructure/**`、`src/taichu/api/schemas/**`、`tests/**`、必要 fixture。

禁止：真实 LLM、真实 RAG、完整前端页面、入口点云页、多小说字段、删除智能体工作台。

具体任务：

1. 分卷章节大纲 JSON：保存 `volume_id`、卷名、卷顺序、`chapter_id`、`display_title`、章节顺序。大纲属于用户创作数据，不是缓存。
2. 章节正文：保存和读取必须保留多行空格、多行换行、连续空行，不得 `trim` 或压缩空白。章节标题只来自左侧大纲，不在正文内编辑。
3. 来源引用：统一结构为来源类型、来源 ID、显示名称、原文摘录、备注。来源类型支持章节、知识卡、作者说明、外部资料。摘录 300 字以内。作者说明必须保存正文。
4. 知识卡 JSON：每张一文件，按类型分目录。公共字段为 `id`、`type`、`name`、`aliases`、`summary`、`body`、`tags`、`importance`、`status`、`source_refs`、`created_at`、`updated_at`。状态为草稿、有效、废弃。直接新建默认草稿，草稿不参与 AI 检索和证据回答，废弃不物理删除。
5. 知识类型：角色、境界、功法、地点、势力、物品、规则、事件、伏笔。
6. 角色字段：身份、重要程度、所属势力、当前境界、功法、物品、关系摘要、出场章节、状态记录。状态记录包含时间点、章节、境界、位置、生死、阵营、备注。不做秘密功能，不做当前目标。
7. Inbox：灵感、待确认事实、待处理问题。状态为待处理、已处理、已废弃。待确认事实确认入库后原记录保留，状态变已处理。
8. `AIWorkspaceConversation`：写作区 AI 对话。根字段包括 `id`、`chapter_id`、`task_type`、`subtask_type`、`reference_scope`、`model_name`、`is_mock`、`source_refs`、`messages`、`created_at`、`updated_at`。消息有独立 `message_id`，助手消息有 `output_type`，每轮保存 `prompt_snapshot`。
9. 设置偏好：字体大小、字体样式、编辑页面背景等基础偏好；不含真实 LLM 配置。

验收命令：`uv run python -m unittest discover tests`、`uv run ruff check .`、`uv run mypy`。完成后停机并按证据模板返回。

## Phase 2：后端 API

目标：实现前端可调用的 MVP API。

允许修改：`src/taichu/api/**`、`src/taichu/application/**`、`src/taichu/domain/**`、`src/taichu/infrastructure/**`、`tests/**`。

禁止：真实 LLM、真实 RAG、写作区 LangGraph、删除智能体工作台、草稿知识卡进入证据回答。

接口建议：

```text
GET    /api/outline
POST   /api/outline/volumes
PATCH  /api/outline/volumes/{volume_id}
POST   /api/outline/chapters
PATCH  /api/outline/chapters/{chapter_id}
GET    /api/chapters/{chapter_id}
PUT    /api/chapters/{chapter_id}
GET    /api/knowledge/types
GET    /api/knowledge/cards?type=&status=&q=
POST   /api/knowledge/cards
GET    /api/knowledge/cards/{id}
PATCH  /api/knowledge/cards/{id}
POST   /api/knowledge/cards/{id}/mark-active
POST   /api/knowledge/cards/{id}/mark-deprecated
GET    /api/inbox?tab=
POST   /api/inbox/ideas
POST   /api/inbox/pending-facts
POST   /api/inbox/issues
PATCH  /api/inbox/ideas/{id}
PATCH  /api/inbox/pending-facts/{id}
PATCH  /api/inbox/issues/{id}
POST   /api/inbox/pending-facts/{id}/confirm
POST   /api/ai-workspace-conversations
POST   /api/ai-workspace-conversations/{id}/messages
GET    /api/ai-history
GET    /api/ai-history/{conversation_id}
GET    /api/settings/preferences
PATCH  /api/settings/preferences
```

关键要求：

1. 知识卡标记有效前校验名称、摘要、来源引用。
2. 废弃知识卡不物理删除。
3. 知识库搜索第一版只在当前类型内。
4. 发送 AI 消息接口返回完整对话。
5. 每轮生成提示词快照。
6. 错误响应包含中文 `message`，前端不得直接展示英文错误码。

验收命令同 Phase 1。完成后停机。

## Phase 3：前端页面

目标：实现主导航和主要页面结构，对接 API 或临时 mock。

允许修改：`web/src/app/**`、`web/src/components/**`、`web/src/lib/**`、`web/src/types/**`、`web/src/app/globals.css`、必要 `web/package.json`。

禁止：修改后端业务、真实 LLM、入口点云页、英文用户文案、游戏 UI、赛博 HUD、普通 SaaS 风。

具体任务：

1. 主导航：写作、知识库、Inbox、AI 历史、智能体工作台、设置。智能体工作台标记“实验功能”。
2. 写作页：左侧分卷章节树和章节摘要，中间正文编辑器，右侧窄栏 AI 入口，入口展开面板。
3. 右侧入口顺序：纯对话、续写、润色、设定、建议、证据、章节摘要、灵感、事实。展开面板压缩编辑区，不覆盖编辑器。一个时间只展开一个入口。
4. 命名体系切换：默认清晰名；品牌化名为问灵、衍文、润笔、构界、策议、溯源、章要、灵引、事实簿。刷新后回到默认清晰名。
5. 编辑器：移除“正文 / 标题一 / 标题二 / 引用”按钮；提供字体大小、字体样式、编辑背景、撤销、恢复、查找、一键段落排版、保存状态、手动保存；不做替换；段落排版只做视觉缩进；保存空白不丢。
6. 左侧大纲：新建卷、新建章节、重命名卷、重命名章节；只展开当前卷；新建卷默认展开；去掉章节显示名前导零；不拖拽排序。
7. 知识库页面：左侧类型导航和筛选，中间列表，右侧详情编辑；可新建知识卡、新建当前类型；新建先前端临时草稿，首次保存写后端；详情按钮为保存、标记为有效、标记废弃；字段名中文。
8. 来源引用：统一“添加来源”弹窗或面板。支持章节、知识卡、作者说明；外部资料预留但不开放填写。
9. Inbox：灵感、待确认事实、待处理问题三类 Tab。待确认事实详情左侧原事实、右侧知识卡预览；确认入库先选知识类型。
10. AI 历史：列表展示时间、章节、功能入口、模型、是否错误、是否有来源、首条用户输入摘要、模拟输出；详情展示消息内容、任务类型、参考范围、输出类型、来源引用、提示词快照；快照默认折叠。
11. 设置页：基础偏好，不显示模型、API Key、服务地址。

验收命令：`cd web && npm run test:editor`、`npm run lint`、`npm run build`。完成后停机。

## Phase 4：假 AI 链路

目标：打通写作区右侧 AI 入口、AIWorkspaceConversation、提示词快照、假 LLM 输出、AI 历史页面。

允许修改：前后端相关 AI 任务链路和测试。

禁止：真实 LLM、真实 RAG、复杂检索模拟、输出质量评测、右侧设定一键转事实、纯对话保存为对话记录、章节摘要进入普通 AI 历史。

具体任务：

1. 纯对话：前端内存临时历史，刷新丢失，不保存。
2. 续写：保存对话，假输出只返回正文候选文本，不解释。
3. 润色：扩写、缩写、改写，输出正文结果。
4. 设定：输出设定补充、使用建议、可能影响；不提供一键转事实。
5. 建议：输出问题、判断、建议。
6. 证据：输出结论、依据、推测、未确认点、模拟来源引用。
7. 章节摘要：不进入普通 AI 多轮记录；左侧显示。本章为空时提示“当前章节为空，无法生成摘要”。没生成过显示“本章暂未生成摘要”。多次生成覆盖。
8. 灵感：手动进入 Inbox 灵感。
9. 事实：手动进入 Inbox 待确认事实。
10. 每轮保存提示词快照：作者输入、正文参考、参考范围、任务类型、子任务类型、skill、route、最终提示词字符串。
11. 前端和 AI 历史显示“模拟输出”。

验收命令：后端测试、ruff、mypy、前端 test/lint/build。完成后停机。
