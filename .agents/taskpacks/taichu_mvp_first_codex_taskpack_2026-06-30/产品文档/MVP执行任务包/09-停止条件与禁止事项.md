# 09-停止条件与禁止事项

> 更新日期：2026-06-30

## 1. 硬停止条件

遇到以下情况必须停止：

1. 必读文件缺失。
2. 最新产品文档与任务包冲突。
3. 需要接真实 LLM 才能继续。
4. 需要实现真实 RAG 才能继续。
5. 需要评测回答质量、模型质量或 RAG 效果。
6. 需要写作区触发 LangGraph。
7. 需要删除智能体工作台。
8. 需要引入多小说管理、`project_id`、`novel_id` 或多租户。
9. 无法保证正文多行空格、多行换行、连续空行保存。
10. 需要把英文内部字段或枚举直接展示给用户。
11. 需要破坏 `start.bat`、uv、Next.js 当前启动方式。
12. 需要在设置页加入真实 LLM 配置。
13. 阶段验收命令失败且无法判断是否由本阶段引起。
14. 需要在停机点前自动 commit。

## 2. 禁止事项

不做多小说平台、多人协作、发布分发、一键自动写书、纯 Agent 技术展示平台。

不接真实 LLM，不接 API Key，不做模型对比，不做自动评分，不做提示词质量评测，不做真实 RAG 效果验证。

不显示“正文 / 标题一 / 标题二 / 引用”四个顶部按钮，不做替换功能，不把首行缩进写成两个全角空格，不压缩作者输入空白，不在正文编辑区直接编辑章节标题。

不物理删除知识卡，不让草稿参与 AI 检索和证据回答，不让废弃卡进入常规全部列表，不让来源为空的知识卡标记有效，不自动把草稿变有效，不做角色秘密功能，不做当前目标字段。

不出现英文按钮、英文功能说明和内部状态文案，不把 `source_refs`、`task_type`、`output_type`、`NON_FACT` 等内部标识直接显示给用户。

## 3. 停止返回格式

```text
停止原因：
触发的停止条件：
已完成检查：
未继续执行的原因：
需要作者决定的问题：
建议选项：
```
