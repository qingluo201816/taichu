# 02-验收清单

> 更新日期：2026-06-30

## 1. 总体验收

- [ ] 主导航依次为：写作、知识库、Inbox、AI 历史、智能体工作台、设置。
- [ ] 智能体工作台保留，并标记“实验功能”。
- [ ] 用户界面没有英文按钮、英文功能介绍、内部枚举裸露。
- [ ] 设置页不显示模型、API Key、服务地址等真实 LLM 配置。
- [ ] 前端遵守 `TAICHU_DESIGN.md`，主工作台不是游戏 UI、赛博 HUD 或普通 SaaS landing page。

## 2. 编辑器

- [ ] 不显示“正文 / 标题一 / 标题二 / 引用”四个顶部按钮。
- [ ] 有字体大小、字体样式、编辑页面背景选择入口、撤销、恢复、查找、一键段落排版、保存状态、手动保存。
- [ ] 第一版没有替换功能。
- [ ] 一键段落排版只做视觉首行缩进，不写入两个全角空格。
- [ ] 多行空格保存、刷新、切换章节后不丢失。
- [ ] 多行换行保存、刷新、切换章节后不丢失。
- [ ] 连续空行不设上限，作者输入多少保留多少。
- [ ] 章节标题只由左侧章节名映射展示，正文编辑区不可直接编辑。
- [ ] `Ctrl+S` 与点击保存按钮等同。
- [ ] 保存状态只显示未保存、保存中、已保存、保存失败。

## 3. 分卷章节大纲

- [ ] 左侧大纲为“分卷 → 章节”。
- [ ] 卷默认显示卷名。
- [ ] 章节列表只展开当前选中卷。
- [ ] 可新建卷。
- [ ] 可新建章节。
- [ ] 可重命名卷。
- [ ] 可重命名章节。
- [ ] 新建章节归属当前卷。
- [ ] 新建卷后默认展开。
- [ ] “第0001章”展示为“第1章”。
- [ ] 第一版不支持拖拽排序。
- [ ] 大纲 JSON 单独保存卷、章节顺序和展示标题，刷新后不丢失。

## 4. 写作区右侧入口

- [ ] 入口顺序：纯对话、续写、润色、设定、建议、证据、章节摘要、灵感、事实。
- [ ] 同一时间只展开一个入口。
- [ ] 展开面板压缩编辑区，不覆盖编辑器。
- [ ] 面板结构为上方正文参考、中间结果记录、底部输入框。
- [ ] 纯对话只保留前端内存历史，刷新丢失，不保存为 AI 多轮对话记录。
- [ ] 续写、润色、设定、建议、证据保存为写作区 AI 多轮对话记录。
- [ ] 章节摘要不进入普通 AI 多轮对话记录，显示在左侧章节摘要区域。
- [ ] 灵感进入 Inbox 灵感。
- [ ] 事实进入 Inbox 待确认事实。
- [ ] 右侧“设定”入口第一版不允许一键转成待确认事实。

## 5. 假 AI 链路

- [ ] 创建对话接口为 `POST /api/ai-workspace-conversations`。
- [ ] 发送消息接口为 `POST /api/ai-workspace-conversations/{id}/messages`。
- [ ] 发送消息接口返回完整对话。
- [ ] 每条消息有独立 `message_id`。
- [ ] 助手消息保存 `output_type`。
- [ ] 写作区结果统一使用 `content` 字段。
- [ ] 每轮保存提示词快照。
- [ ] 假输出显示“模拟输出”。
- [ ] AI 历史页面显示“模拟输出”标记。
- [ ] AI 相关验收只检查输出结构、链路和提示词快照，不检查真实回答质量。

## 6. Inbox

- [ ] Inbox 按灵感、待确认事实、待处理问题三类 Tab 展示。
- [ ] 灵感字段为内容、来源章节、优先级。
- [ ] 写作区“事实”入口字段为标题、内容、来源章节、出处、优先级。
- [ ] 状态由系统默认设为待处理，不作为作者必填项。
- [ ] 待确认事实详情为左侧原事实、右侧知识卡预览。
- [ ] 待确认事实确认入库时必须先选择知识卡类型。
- [ ] 确认入库成功后，原 Inbox 记录保留，状态变为已处理。

## 7. 知识库

- [ ] 知识库是一级页面。
- [ ] 页面布局为左侧类型导航和筛选、中间知识卡列表、右侧详情编辑区。
- [ ] 类型包括角色、境界、功法、地点、势力、物品、规则、事件、伏笔。
- [ ] 左侧类型导航顶部有“新建知识卡”。
- [ ] 当前类型列表顶部有“新建当前类型”。
- [ ] 新建知识卡时先在前端生成临时草稿，首次保存再写后端。
- [ ] 直接新建知识卡默认草稿。
- [ ] 草稿不参与 AI 检索和证据回答。
- [ ] 知识卡标记有效前必须校验名称、摘要和来源引用。
- [ ] 草稿补齐来源后不会自动变有效，必须作者点击“标记为有效”。
- [ ] 知识卡不物理删除，只能标记废弃。
- [ ] 废弃默认隐藏，通过“只看废弃”查看。
- [ ] 状态筛选为全部、草稿、有效；废弃单独查看。
- [ ] 知识库搜索只在当前选中知识类型内搜索。
- [ ] 来源引用结构统一：来源类型、来源 ID、显示名称、原文摘录、备注。
- [ ] 来源摘录限制 300 字以内。
- [ ] 作者说明来源需要独立保存正文内容。

## 8. AI 历史

- [ ] AI 历史是一级页面。
- [ ] 列表展示时间、章节、功能入口、模型、是否错误、是否有来源、首条用户输入摘要、模拟输出标记。
- [ ] 筛选包括章节、功能入口、是否有来源、是否有错误。
- [ ] “模拟输出”只显示为标记，不作为筛选条件。
- [ ] 详情显示消息内容、本轮任务类型、参考范围、输出类型、来源引用、提示词快照。
- [ ] 提示词快照默认折叠。

## 9. 测试重点

- [ ] 后端测试覆盖保存、空白保留、卷章大纲、知识卡创建/读取/更新/标记有效/标记废弃、Inbox、假 AI 链路。
- [ ] 前端测试覆盖编辑器空白保留、分卷章节树、右侧 AI 入口、知识库状态、Inbox 确认入库、AI 历史提示词快照。
- [ ] 全部可用命令执行并记录结果。
